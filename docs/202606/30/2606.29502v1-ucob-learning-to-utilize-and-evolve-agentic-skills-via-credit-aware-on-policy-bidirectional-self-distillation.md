## 摘要
技能记忆可以通过将过去经验作为文本指导重用来改善智能体强化学习，但检索到的技能并非全知全能：它们可能在一种状态下有帮助，但在另一种状态下误导同一策略。这使得常见的特权教师假设变得脆弱，即技能条件提示可以被视为无技能提示的固定教师。我们提出UCOB，一个通过信用感知的在策略双向自蒸馏学习利用和演化智能体技能的框架。UCOB将技能条件和无技能提示视为同一模型的两个在策略上下文视图，比较它们在相同任务和锚定状态下的预期回报，并使用更高回报的视图作为局部教师。这种局部信用信号内化了有用的技能条件行为，纠正了误导性的技能使用，并指导任务/状态技能记忆更新、效用感知检索和反思自训练。在包括ALFWorld、WebShop和Search-QA在内的智能体任务上的实验表明，UCOB在不同模型规模上优于无技能RL、技能记忆基线和自蒸馏方法，在ALFWorld和WebShop上比最先进的基线分别高出23.5和18.0个点。消融和分析进一步验证了其核心机制和效率。

## Abstract
Skill memories can improve agentic reinforcement learning by reusing past experience as textual guidance, but retrieved skills are not oracular: they may help in one state while misleading the same policy in another. This makes the common privileged-teacher assumption fragile, namely that a skill-conditioned prompt can be treated as a fixed teacher for the no-skill prompt. We introduce UCOB, a framework for learning to utilize and evolve agentic skills via credit-aware on-policy bidirectional self-distillation. UCOB treats skill-conditioned and no-skill prompts as two on-policy context views of the same model, compares their return-to-go within the same task and anchor state, and uses the higher-return view as the local teacher. This local credit signal internalizes useful skill-conditioned behavior, corrects misleading skill usage, and guides task/state skill memory updates, utility-aware retrieval, and reflection self-training. Experiments on agentic tasks, including ALFWorld, WebShop, and Search-QA, show that UCOB outperforms skill-free RL, skill-memory baselines, and self-distillation methods across model scales, with up to 23.5 and 18.0 point gains over SOTA baselines on ALFWorld and WebShop. Ablations and analyses further validate its core mechanisms and efficiency.

## 1. 论文的核心问题与整体含义（研究动机和背景）

**研究动机**：在智能体强化学习（Agentic RL）中，利用过往经验形成的技能记忆（Skill Memory）作为文本提示，可以显著提升学习效率。然而，检索到的技能并非总是有益的：同一个技能可能在一个状态下帮助策略，却在另一个状态下误导同一策略。现有方法（如SDAR、Skill-SD）普遍采用“特权教师假设”（privileged-teacher assumption），即把技能条件化的提示视为固定教师，将知识单向蒸馏给无技能提示。该假设脆弱且不可靠——技能条件化分支并非始终优于无技能分支，即使让技能分支进行在策略回滚也无法消除这种歧义。

**整体含义**：论文提出应当抛弃固定教师视角，转而将技能条件和无技能提示视为同一在线策略的两个上下文视图。通过比较同一任务、同一锚定状态（anchor state）下的预期回报，选择更高的回报视图作为局部教师，从而内化有用技能行为、纠正误导性技能使用，并进一步指导技能记忆的更新、检索和技能编写者的自训练。该框架名为UCOB（learning to Utilize and evolve agentic skills via Credit-aware On-policy Bidirectional self-distillation）。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
UCOB通过在线策略双向自蒸馏（Credit-Aware Bidirectional Self-Distillation, CBSD）实现技能利用与演化的闭环。它将技能条件提示和无技能提示视为同一模型的两个上下文视图，不预先固定教师方向，而是通过同一锚定状态组内的回报比较，动态选择高回报视图作为教师。

### 关键技术细节

- **双粒度技能记忆与混合回滚**：UCOB维护任务级和状态级两个技能池，并在线扩展。技能包含文本原则、适用条件、检索键、效用分数和计数。检索时采用基于置信上界（UCB）的排序，综合考虑语义相似性、效用值和探索项。每次训练批量中，一半轨迹使用技能条件提示，另一半使用无技能提示，交互后获得回报。

- **锚定状态分组**：按照任务和锚定状态（anchor state）将回滚记录分组，锚定状态是提取状态中的规范化抽象表示。同一组内包含来自两个分支的记录，每条记录保存状态、分支标签、所用提示、对应分支提示、动作和预期回报。

- **信用感知双向自蒸馏（CBSD）**：
  - 对于每个锚定状态组，选择预期回报最高的记录作为“信用参考”（credited reference），其所在分支作为教师方向。
  - 如果技能分支获胜，则技能→无技能；如果无技能分支获胜，则无技能→技能。只有当回报差距大于阈值时才保留蒸馏对。
  - 蒸馏时采用Top-K令牌支持匹配（Top-K token-support matching）：在参考分支的前缀下计算令牌分布，取前K个令牌并重归一化，然后计算KL散度作为损失。
  - 引入置信门（confidence gate）和基于回报差距的尺度权重，使蒸馏强度与局部信用信号成正比。

- **技能记忆演化**：
  - 任务级技能通过回滚组反思生成；状态级技能通过同一锚定状态下高/低回报记录的对比生成。
  - 技能效用通过指数移动平均更新，基于使用该技能后相对于无技能基线的成功率提升。
  - 技能编写者（skill writer）自身进行自训练：利用效用计算出的优势值，采用策略梯度优化编写行为。

- **整体损失函数**：
  L = L_RL + λ_CBSD L_CBSD + λ_refl L_refl + β_KL L_KL
  其中L_RL为组内分组策略梯度损失（基于GiGPO），L_CBSD为双向蒸馏损失，L_refl为技能编写者反思损失，L_KL为参考策略正则化。

- **理论支撑**：论文通过命题1-4证明，在正确估计分支顺序和局部信任区域条件下，CBSD可提供局部策略改进方向，避免固定教师方向下的负向更新。

## 6. 论文的主要结论与发现

1. **技能条件化教师不可靠**：通过诊断实验（图3、图4、图5）证实，即使进行在策略回滚，技能条件化分支并不始终优于无技能分支，教师方向应根据局部回报差距动态选择。
2. **CBSD有效提升性能**：在ALFWorld上，UCOB在Qwen3-1.7B上达到89.1%成功率，超过Skill1（65.6%）和SDAR（53.9%）；在WebShop上达到79.7%成功率，超过Skill1（61.7%）和SDAR（58.6%）。最大提升幅度分别为+23.5和+18.0点。
3. **消融实验揭示了关键组件**：去除CBSD导致显著性能下降，说明自蒸馏信号不可或缺；去除混合回滚也造成较大损失，表明双视图在线回滚是信用比较的基础。状态级技能和反思自训练提供互补增益。
4. **CBSD路由大多为无技能→技能**：统计显示在锚定状态下无技能分支获胜率更高，说明技能提示并非更优，正确的教师方向往往是“无技能教师”。
5. **技能记忆保持有用性**：随着训练进行，检索到的技能效用维持正值，有用技能比例稳定，反思优势信号保持活跃。
6. **额外开销适度**：CBSD带来的训练时间增加主要是策略回滚，其他计算（旧logprob、参考logits等）只占小部分。

## 8. 不足与局限

- **Search-QA上表现并非压倒性**：在搜索问答场景中UCOB的平均准确率与最佳基线相当甚至略低（如Qwen2.5-7B上为48.2% vs 某基线49.0%），说明双向蒸馏在单步或少步交互任务中增益有限，可能因为技能信用信号不够稳定。
- **计算开销仍有提升空间**：虽然CBSD自身开销不大，但混合回滚（双视图各一半轨迹）意味着每次训练需要生成双倍轨迹，训练成本约为无技能RL的两倍；论文未详细对比总训练时间绝对值，仅展示了相对效率。
- **理论分析有待深化**：提供的定理（Proposition 1-4）聚焦于单步局部改进，并未给出整个非平稳闭环系统的收敛性保证；论文自己也标注“a local, one-update interpretation, not a global convergence claim”。
- **依赖锚定状态抽象**：锚定状态的定义对比较结果敏感，论文未详细讨论不同抽象粒度或噪声对性能的影响，可能在实际应用中需要专门设计。
- **技能种类和生成质量**：技能通过反思生成，其质量取决于反思模板和模型初始能力；若反思产生低质或重复技能，可能污染记忆库。论文通过自训练部分缓解，但未分析技能多样性或冗余度。
