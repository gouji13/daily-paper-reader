## 摘要
经验驱动的自我进化对于大型语言模型（LLM）智能体通过开放世界互动进行改进至关重要。然而，现有的经验学习方法大多依赖于单一智能体循环，即同一智能体执行任务、总结结果并确定记忆内容。这种设置使智能体容易陷入“自我确认陷阱”：错误但自洽的轨迹被误判为成功经验，导致在检索和复用过程中累积错误。为解决此问题，我们提出EDV，一个用于可靠经验学习的“执行-蒸馏-验证”框架。在执行阶段，多个异构智能体并行探索相同的任务空间，生成多样化的候选轨迹。在蒸馏阶段，一个专门的第三方智能体对这些轨迹进行比较分析以产生候选经验，减少以执行者为中心的总结偏差。在验证阶段，执行组通过共识机制验证候选经验，只有通过的经验被写入共享或私有记忆。通过解耦三个阶段，EDV将经验学习从孤立的自我反思转变为协作构建，在记忆插入前过滤错误和噪声内容。我们在三个具有挑战性的长时域基准上评估EDV：tau2-bench、Mind2Web和MMTB。结果表明EDV始终优于强基线，验证了可靠经验构建对于鲁棒的智能体自我进化至关重要。我们的代码可在https://github.com/shidingz/EDV获取。

## Abstract
Experience-driven self-evolution is critical for large language model (LLM) agents to improve through open-world interaction. However, existing experience learning methods mostly rely on single-agent loops, where the same agent executes tasks, summarizes outcomes, and determines memory content. This setup makes agents vulnerable to the Self-Confirmation Trap: wrong-but-self-consistent trajectories are misidentified as successful experience, leading to cumulative errors during retrieval and reuse. To address this issue, we propose EDV, an Execute-Distill-Verify framework for reliable experience learning. In the Execute stage, multiple heterogeneous agents explore the same task space in parallel to generate diverse candidate trajectories. In the Distill stage, a dedicated third-party agent comparatively analyzes these trajectories to produce candidate experiences, reducing executor-centric summarization bias. In the Verify stage, the execution group validates candidates via a consensus mechanism, and only approved experiences are written into shared or private memory. By decoupling the three stages, EDV transforms experience learning from isolated self-reflection into collaborative construction, filtering erroneous and noisy content before memory insertion. We evaluate EDV on three challenging long-horizon benchmarks: tau2-bench, Mind2Web and MMTB. Results show EDV consistently outperforms strong baselines, validating that reliable experience construction is essential for robust agent self-evolution. Our code is available at https://github.com/shidingz/EDV.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）智能体在开放世界环境中需要通过经验学习持续自我改进。然而，现有方法大多采用**单智能体循环**——同一智能体既执行任务又评估结果，导致容易陷入**自我确认陷阱（Self-Confirmation Trap）**：错误但自洽的轨迹被误判为成功经验，写入记忆后通过检索与复用逐步放大误差。
- **整体意义**：经验学习的瓶颈不在于收集更多经验，而在于**如何更可靠地构建经验**，在进入记忆前抑制错误和噪声内容。本文提出解耦式协作框架，将经验构建从孤立反思转变为多智能体过滤与验证，从而提升自我进化的鲁棒性。

## 2. 方法论：核心思想、关键技术细节

- **框架名称**：EDV（Execute-Distill-Verify）
- **核心思想**：通过**异构多智能体协作**将经验学习的三个环节（执行、蒸馏、验证）解耦，避免单智能体耦合导致的偏见。

- **三个阶段**：
  1. **Execute（执行）**：从异构智能体池（例如 MiniMax-M2.1、Mimo-V2-Flash、GLM-4.7-FP8）中随机抽取两个智能体组成执行组，每个智能体独立与任务环境交互，生成候选轨迹集合 \( T(q) = \{\tau_i\} \)。目的是扩大解空间覆盖、暴露多样的成功和失败模式。
  2. **Distill（蒸馏）**：从池中随机选择第三个智能体作为第三方蒸馏智能体，对执行组产生的多条轨迹进行比较分析，提取跨轨迹的**候选经验集合** \( E_{\text{cand}} \)。蒸馏智能体不重复单条轨迹，而是标识多轨迹之间的有用差异。
  3. **Verify（验证）**：执行组的原始智能体对每个候选经验进行二元验证 \( V_j(e) \in \{0,1\} \)。采用**严格默认拒绝策略**：
     - 全票通过 → 写入**共享记忆库**；
     - 部分通过 → 写入对应智能体的**私有记忆库**；
     - 全票拒绝 → 丢弃。
- **附加组件**：维护一个**能力矩阵（Ability Matrix）**记录各智能体擅长的任务类型；推理时先用能力矩阵选择最佳求解器，然后层次化检索（先共享库，若不充分再查私有库）。

- **简要流程**：异构执行 → 第三方蒸馏 → 共识验证 → 存储 → 推理时能力匹配 + 分层检索。

## 3. 实验设计

  - **Mind2Web**（网页交互）：包含跨任务、跨网站、跨域三种泛化设置，评估元素选择、操作F1、步骤成功率、任务成功率。
  - **MMTB**（多工具任务）：评估单动作、多动作（串行/并行/混合）、轨迹路径等层次化指标。

  - **RB（ReasoningBank）**：代表性单智能体记忆学习方法。

## 6. 主要结论与发现

1. **EDV 在所有基准上一致超越强基线**。τ2-bench 平均 Pass@1 达 86.6，大幅高于 Router（83.5）和 Judge（81.5）；Mind2Web 和 MMTB 上也持续领先。

4. **解耦设计是关键**：单智能体中即使加入自检也无法修复陷阱；多智能体异质性 + 第三方蒸馏 + 共识验证三者协同产生最大增益。
5. **能力矩阵和分层记忆均不可或缺**：去掉能力矩阵下降2.0分，去掉任一记忆层下降2.7-2.9分；私有记忆覆盖31.8%的任务并带来1.8%的单次增益，显示其对个性化边缘案例的支持。
6. **效率与效果兼得**：EDV 将长程推理成本从在线搜索转移到离线高质量经验构建，在 RETAIL 上比 RB 减少24.5%推理 token 消耗。

## 7. 优点

1. **问题洞察深刻**：首次系统提出并形式化“自我确认陷阱”，揭示了单智能体经验学习的根本缺陷。
2. **设计巧妙**：通过异构执行增加轨迹多样性、第三方蒸馏消除执行者偏见、共识验证提高记忆插入门槛，三者环环相扣。

## 8. 不足与局限

- **共识偏差风险**：若异构智能体共享某种失败模式，共识机制可能无意中验证噪声（附录G）。
