## 摘要
对于在多步交互环境中的大型语言模型代理，一个关键挑战是如何有效利用积累的交互经验。现有工作通常将这种经验的两类用途分开：将经验作为自然语言规则保留在模型之外以便后续提示，或者使用轨迹和反馈来更新模型参数。前者易于解释，但可能随着策略的演变而失去同步；后者更广泛地改进了策略，但在稀疏奖励设置中仅对局部错误提供有限的纠正。我们提出了面向LLM代理的经验规则与策略联合学习（JERP），该方法从相同的交互轨迹中更新长期经验规则池和策略。在决策时，JERP检索与任务相关的规则，并将这些规则与交互历史一起作为代理的条件。每次情节结束后，它利用收集到的轨迹来优化策略，并通过将当前展开与参考成功轨迹进行比较来修订规则池。这种耦合使得规则池与不断演变的策略保持一致，同时允许稳定有效的行为逐渐被吸收到模型本身中。在AlfWorld和WebShop上的实验表明，JERP在复杂交互任务的决策性能上取得了一致的提升。

## Abstract
For LLM agents in multi-step interactive environments, a key challenge is to make effective use of accumulated interaction experience. Existing work has typically separated two uses of such experience: keeping it outside the model as natural-language rules for later prompting, or using trajectories and feedback to update the model parameters. The former is easy to interpret but can fall out of sync with the evolving policy; the latter improves the policy more broadly but provides only limited correction for local mistakes in sparse-reward settings. We present Joint Learning of Experiential Rules and Policies for LLM Agents (JERP), which updates a long-term experiential-rule pool and the policy from the same interaction trajectories. At decision time, JERP retrieves task-relevant rules and conditions the agent on them together with the interaction history. After each episode, it uses the collected trajectories both to optimize the policy and to revise the rule pool by comparing current rollouts with reference successful trajectories. This coupling keeps the rule pool aligned with the evolving policy while allowing stable and effective behaviors to be gradually absorbed into the model itself. Experiments on AlfWorld and WebShop show that JERP yields consistent gains in decision performance for complex interactive tasks.

## 1. 核心问题与整体含义

- **研究动机**：大型语言模型（LLM）在多步交互环境（如网页导航、具身交互、工具使用）中需要有效利用积累的交互经验，但现有方法将经验的两类用途分离：要么将经验作为自然语言规则存储在模型外部（如反射性文本、指导规则），要么使用轨迹和反馈来更新模型参数（如强化学习微调）。这两类方法各有局限：外部规则易于解释，但可能随策略演变而失去同步；参数更新能全局改进策略，但在稀疏奖励设置下对局部错误的纠正有限。
- **核心问题**：能否让同一批交互数据同时支持显式规则更新和策略优化，使两者相互协调、共同进化？
- **整体含义**：JERP通过联合学习经验规则池和策略参数，使得规则池与不断演变的策略保持对齐，同时让稳定有效的行为逐渐被吸收到模型本身中，从而在复杂交互任务上获得一致的性能提升。

## 2. 方法论

- **核心思想**：JERP维护一个长期经验规则池（每个任务实例对应一个），并在每个情节中选取一个工作规则子集与交互历史一起构成决策上下文。情节结束后，同一批轨迹被用于两个目的：一是通过组相对策略优化（GRPO-like）更新模型参数，二是通过对比当前轨迹与参考成功轨迹来更新规则池。
- **关键技术细节**：
  - **规则池表示**：对每个任务实例 d，规则池 K(d) = {(z_i, s_i)}，其中 z_i 为规则文本，s_i 为效用分数。
  - **工作规则选择**：按效用分数降序选择 top-k 条规则作为当前情节的工作规则集 eK(d)，情节内固定。
  - **策略生成**：在每一步 t，输入提示 x_t = (d, h_t, eK(d))，策略 πθ 采样动作 a_t。
  - **策略更新**：采样 N 条完整轨迹，计算每条轨迹的最终奖励 R(τ_i)，在组内标准化得到优势 A_i = (R(τ_i) - μ_d) / (σ_d + δ)，所有时间步共享该优势。使用带裁剪的 GRPO 目标函数更新策略参数，包含 KL 正则化。
  - **规则更新**：通过对比反射（ReflectAndEdit）生成五种规则编辑操作：ADD（添加规则）、EDIT（修改规则）、UPVOTE（增加效用分）、DOWNVOTE（降低效用分）、MERGE（合并语义相似的规则）。规则效用分采用固定步长更新，低于阈值则删除。同时利用参考成功轨迹集 T⁺(d) 辅助更新。
- **算法流程**（文字说明）：
  1. 初始化每个任务实例的规则池和参考成功轨迹集为空。
  2. 每轮训练：采样一个任务实例 d，将旧策略设为当前策略，从规则池选出 top-k 工作规则。
  3. 用当前策略执行 N 个情节，收集轨迹组 T_d。
  4. 计算组内优势，用 GRPO 目标更新策略参数。
  5. 调用 ReflectAndEdit 从规则池、T_d 和参考成功轨迹生成规则编辑操作，并应用更新规则池。
  6. 更新参考成功轨迹集（加入奖励大于 0 的轨迹）。
  7. 重复至训练步数。

## 6. 主要结论与发现

- JERP在AlfWorld总体成功率达到61.5%，优于GRPO（57.8%）和RLOO（48.7%）；在WebShop上平均分数79.0，成功率64.1%，同样优于所有基线。

- 结论：耦合规则池与策略演化使LLM代理在复杂交互任务中能稳定学习并逐步内化有效行为。
