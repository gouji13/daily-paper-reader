## 摘要
基于可验证奖励的强化学习（RLVR）利用稀疏的可验证最终答案奖励来优化大语言模型。这种稀疏锚点可靠地验证轨迹是否成功，但对产生该轨迹的推理路径不提供直接反馈。在成功之前，解决难题所需的先决进展得不到任何奖励信号；在成功之后，结果奖励无法区分组织良好的正确轨迹与冗余或局部有缺陷的轨迹。我们提出SCOPE-RL（带过程效率的支架链优化），这是一个两阶段框架，它在保留GRPO更新的同时使该锚点变密集：自适应支架RL在成功之前对隐藏答案的子问题链添加前缀分解的可验证奖励，而质量感知过程RL在成功之后应用正确性门控的过程形状奖励来细化正确轨迹。一个经专家验证的步骤质量评估协议评估有用步骤密度、错误定位和令牌效率，超越了最终答案准确率。在基于DAPO-Math和Big-Math训练的Qwen3-8B-Instruct上，SCOPE-RL相比仅结果GRPO平均准确率提升高达11.2个百分点，推理令牌减少高达27.1%；这些收益在GSPO和Qwen3-0.6B-Instruct上仍然保持，表明奖励信号密化与策略更新层面的RLVR进展互补。代码和数据可在https://github.com/tokencraft-lab/SCOPE-RL获取。

## Abstract
Reinforcement learning with verifiable rewards (RLVR) optimizes LLMs using sparse verifiable final-answer rewards. This sparse anchor reliably verifies whether a trajectory succeeds but provides no direct feedback on the reasoning path that produced it. Before success, prerequisite progress on hard problems receives no reward signal; after success, outcome rewards cannot distinguish well-organized correct trajectories from redundant or locally flawed ones. We introduce SCOPE-RL (Scaffolded Chain Optimization with Process Efficiency), a two-stage framework that densifies this anchor while retaining the GRPO update: Adaptive Scaffolded RL adds prefix-decomposed verifiable rewards on answer-hidden sub-question chains before success, and Quality-Aware Process RL applies correctness-gated process-shape rewards to refine correct trajectories after success. An expert-validated Step-Quality Evaluation Protocol evaluates useful-step density, error localization, and token efficiency beyond final-answer accuracy. On Qwen3-8B-Instruct trained on DAPO-Math and Big-Math, SCOPE-RL improves average accuracy by up to 11.2 pp and reduces reasoning tokens by up to 27.1% over outcome-only GRPO; the gains hold under GSPO and on Qwen3-0.6B-Instruct, indicating that reward-signal densification is complementary to policy-update-level RLVR advances. Code and data are available at https://github.com/tokencraft-lab/SCOPE-RL.

# 论文总结：SCOPE-RL: Optimizing Reasoning Paths Before and After Success

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：基于可验证奖励的强化学习（RLVR）使用稀疏的最终答案奖励，该奖励虽然可靠地验证轨迹是否成功，但对**推理路径**不提供直接反馈，导致两个失效模式：
  - **成功之前**：难题上很少成功的轨迹，监督信号稀疏，模型无法获得中间进步的奖励。
  - **成功之后**：结果奖励无法区分正确但冗余、局部有缺陷的轨迹与高质量简洁轨迹。
- **动机**：现有方法分别处理这两个问题（探索导向、效率导向、过程监督），但未系统地将规则可验证信号本身通过**成功前和成功后两个阶段**进行密化（densify），同时保留GRPO更新。
- **整体含义**：提出一个两阶段框架SCOPE-RL，在保持GRPO后端不变的前提下，通过**奖励信号密化**来提升学习效率与推理质量，且证明该密化与策略更新层面的RLVR进展互补。

## 2. 方法论

### 核心思想
- 将稀疏的最终答案奖励在**成功前**和**成功后**分别密化：
  - **成功前**（Stage 1）：引入“自适应脚手架RL”（Adaptive Scaffolded RL, ASR），通过可验证的隐藏答案子问题链提供前缀分解奖励，奖励中间进步。
  - **成功后**（Stage 2）：引入“质量感知过程RL”（Quality-Aware Process RL, QPR），对正确的轨迹使用正确性门控的过程形状奖励，鼓励简洁且组织良好的推理。
- 两个阶段**顺序耦合**：Stage 1提升成功率，为Stage 2提供足够的正确轨迹；Stage 2在Stage 1基础上优化过程质量。

### 关键技术细节

#### Stage 1：自适应脚手架RL (ASR)
- **离线构建隐藏答案的脚手架**：对每个训练问题，自动生成子问题链 `S(q) = (q1,a1),...,(qm,am),(q_main,a)`，子答案仅用于奖励计算且对模型隐藏，满足可验证性、前提相关性、依赖性和无答案泄露的谓词约束。
- **On-policy 脚手架路由**：对每个prompt，先用原始prompt采样G个轨迹，计算平均结果奖励 `r̄(q)`。若 `r̄(q) < τ`（阈值τ=0.5），则替换为脚手架prompt重新采样；否则保留原始轨迹和结果奖励。
- **前缀一致性奖励**：对于被路由的组，定义前缀一致性指示器 `Π_i = ∏_{j=1}^i 1[ˆa_j = a_j]`，奖励公式：  
  `R_ASR = β * (1/m) * ∑ Π_i + (1-β) * Π_m * 1[ˆa_main = a]`，其中β=0.5。  
  该奖励在子答案正确时给予部分信用，但最终主答案奖励仅在所有子答案正确时才生效，防止猜测。

#### Stage 2：质量感知过程RL (QPR)
- **正确性门控**：若最终答案错误，奖励为0；只有正确轨迹才进入过程评估。
- **步骤级类别标注**：使用固定LLM Judge（GPT-4.1-mini）将正确轨迹分解为N个原子步骤，并为每步分配互斥标签：`useful`, `mechanical`, `redundant`, `reversion`, `error`。
- **质量感知奖励组成**：  
  `Q_process = S_u · Φ · κ`，其中：
  - `S_u = (1/N) * ∑ 1[c_i = useful]` 为有用步比率。
  - `Φ = ∏_{k∈K} (1 - λ_k * r_k)`，对四种负面步骤（机械、冗余、回退、错误）施加惩罚，λ∈[0,1)。
  - `κ = 1/(1 + α * ln(N+1))` 对长度施加软约束，α=0.5。
- **优势计算**：QPR奖励直接代替结果奖励进入GRPO的组内优势计算。

### 算法流程（文字说明）
1. 初始化策略π_θ。
2. **Stage 1**：  
   - 对每个prompt采样G个原始轨迹，计算平均结果奖励。  
   - 若平均奖励 < τ，则重采样脚手架提示并计算R_ASR；否则保持结果奖励。  
   - 合并所有组，用GRPO更新策略。
3. **Stage 2**：  
   - 从Stage 1最佳检查点初始化策略。  
   - 对原始prompt采样G个轨迹。  
   - 对每个轨迹：若答案正确，用LLM Judge标注步骤并计算Q_process；否则奖励=0。  
   - 使用R_QPR作为标量奖励进行GRPO更新。

## 6. 主要结论与发现

- **准确率显著提升**：SCOPE-RL在DAPO-Math上平均准确率从55.80%提升到66.35%（+10.55 pp），在Big-Math上从53.58%提升到64.79%（+11.21 pp）。
- **推理效率提升**：平均令牌数减少16.2%（DAPO-Math）至27.1%（Big-Math），同时保持或提升有用步密度。
- **过程质量改善**：有用步比率（Useful%）从56.79%升至64.52%（DAPO-Math），错误位置后移。
- **鲁棒性验证**：GSPO后端下准确率从61.60%升至66.93%，令牌减少13.1%；Qwen3-0.6B上准确率从26.06%升至32.06%，令牌减少22.9%。
- **消融揭示关键组件**：
  - ASR需要**前缀一致性奖励**和**on-policy路由**两者共同作用。
  - QPR的**步骤级分解**优于整体评分；**三个组成项（正项、惩罚、长度因子）** 均不可或缺；**顺序训练**优于混合训练。
- 奖励信号密化与策略更新层面的RLVR进展（如GSPO）互补，可进一步叠加。

## 7. 优点

1. **新颖的问题分解视角**：将RLVR的稀疏奖励问题明确拆解为“成功前”和“成功后”两个阶段，并针对性设计密化策略。
2. **保持GRPO后端不变**：仅修改奖励信号，不与现有优化器冲突，易于集成。
3. **实用的脚手架构造**：自动离线生成可验证子问题链，通过严格谓词保证无答案泄露，且通过人类审计验证。
4. **正确性门控的精心设计**：QPR的过程奖励仅作用于正确轨迹，避免误导模型生成简短错误答案。
5. **质量评估协议完备**：提供了超越准确率的细粒度过程诊断（有用步率、错误定位、步骤类别分布），且经专家验证与人类判断高度一致。
6. **实验充分与客观**：大规模消融、跨数据源、跨模型尺度、跨优化器、专家偏好评估，多重验证。
7. **代码与数据开源**：促进可复现性和后续研究。

## 8. 不足与局限

1. **领域局限**：实验限于Qwen族模型和数学/科学推理领域，未覆盖其他模型族（如Llama、Gemini）或更广泛的可验证奖励领域（如编程、逻辑推理）。
2. **方法比较不全**：未与所有基于提示的探索方法（如EvoCoT、POPE、StepHint）或基于长度感知的压缩方法进行详尽的端到端对比，作者承认这是有意范围选择。
3. **超参数调优不充分**：仅对路由阈值τ进行了敏感性分析（附录K），而对β、α、{λ_k}等超参数未做全面扫描，可能未达最优。
4. **Pass@k的轻微退化**：在AIME 2025-I上，SCOPE-RL的Pass@128相比ASR有所下降（80% vs 93.33%），作者解释为质量精炼集中在主流路径而牺牲部分多样性。
5. **训练计算量未明确报告**：缺乏总GPU小时数等资源消耗信息，不利于计算成本比较。
6. **门控依赖最终答案的正确性**：若验证器存在误报，QPR可能会对错误轨迹奖励，但文中未讨论此风险。
