## 摘要
推理上下文长度与强化学习后训练之间的差距日益扩大：推理系统正接近百万token上下文，而后训练工作负载通常仍停留在256K token以下，并依赖部署时的长度泛化。这一差距对于AI智能体尤其重要，其观察、工具输出、文档和先前的决策会在长轨迹中累积。LongStraw是一个面向架构的执行栈，用于在固定GPU预算下进行百万token的强化学习后训练，并以分组相对策略优化（GRPO）实例化。它评估无需自动求导的共享提示，仅保留后续token所需的特定模型状态，并逐个重放短响应分支，以减少实时训练图，代价是增加重放时间。我们为混合循环和全注意力模型Qwen3.6-27B以及压缩注意力混合专家模型GLM-5.2实现了该方案。在八块H20 GPU上，LongStraw在2.1M位置完成了分组Qwen评分和响应反向传播（分组大小为2和8）；增加分组大小仅增加0.21 GB的峰值分配内存，而单独的压力测试达到4.46M位置。在32块H20 GPU上，我们验证了GLM-5.2所有78层上2.1M token提示的端到端LongStraw执行路径。这些实验建立了执行能力而非完整的训练正确性，因为捕获的提示状态是分离的，且某些分布式前向和梯度组合路径仍不完整。

## Abstract
A growing gap separates inference context lengths from RL post-training: inference systems are approaching million-token contexts, while post-training workloads often remain at 256K tokens or below and rely on length generalization at deployment. The gap is especially important for AI agents, whose observations, tool outputs, documents, and prior decisions accumulate over long trajectories. LongStraw is an architecture-aware execution stack for million-token RL post-training under a fixed GPU budget, instantiated with Group Relative Policy Optimization (GRPO). It evaluates the shared prompt without autograd, retains only model-specific state needed by later tokens, and replays short response branches one at a time, reducing the live training graph at the cost of additional replay time. We implement it for the hybrid recurrent and full-attention Qwen3.6-27B and the compressed-attention mixture-of-experts GLM-5.2. On eight H20 GPUs, LongStraw completes grouped Qwen scoring and response backward at 2.1M positions for groups of 2 and 8; increasing the group size adds only 0.21 GB of peak allocated memory, while a separate stress test reaches 4.46M positions. On 32 H20 GPUs, we validate the end-to-end LongStraw execution path for a 2.1M-token prompt across all 78 layers of GLM-5.2. These experiments establish execution capacity rather than complete training correctness because the captured prompt state is detached and some distributed forward and gradient composition paths remain incomplete.

## 1. 核心问题与整体含义（研究动机和背景）
- **长上下文训练的差距**：推理系统已可支持近百万token上下文，但强化学习后训练（如GRPO）通常仍局限于256K token以内，部署时依赖长度泛化。这严重制约了AI智能体的能力——智能体需要处理长轨迹中的观察、工具输出、文档和先前决策。
- **内存瓶颈**：与推理不同，训练需要为同一历史上下文多次评分并反向传播。二次注意力计算和长寿命的 backward 状态使GPU内存成为扩展训练上下文的瓶颈。现有方法（如LoRA、FlashAttention、Ring Attention等）要么仅优化某一环节，要么依赖扩大集群规模。
- **问题聚焦**：LongStraw问的是：**在GPU数量固定不变的情况下，GRPO执行路径能走多远？** 它旨在降低长上下文后训练的硬件门槛，使更多研究者和团队能在有限加速器预算下探索该方向。

## 2. 论文提出的方法论
### 核心思想
- **分离长提示与短响应**：将共享的、超长提示在前向时**不开启自动微分**（no-grad）执行一次，仅保留架构特定的“条件状态”（如循环状态、KV页、MLA潜变量、稀疏索引键等），而瞬时的激活张量立即释放。
- **串行重放响应分支**：在提示状态只读的前提下，逐条处理每个响应分支：先计算旧策略/参考策略的对数概率（冻结），再开启自动微分执行当前策略的前向、反向，累积梯度，然后释放该分支的图。组内所有成员共享同一个提示状态，但梯度累积后只调用一次优化器。
- **将训练图从“(提示+全部响应)”压缩为“一个响应分支”**，用时间换取峰值内存的显著下降。

### 关键技术细节
- **针对Qwen3.6-27B**（混合循环GDN + 全注意力，稠密FFN）：
  - 提示状态：48层GDN的循环状态（固定大小） + 16层全注意力的紧凑KV页（按上下文并行度CP8分片，每rank只保留自有页并复制为独立张量）。
  - 响应重放：将响应分为4个块，逐块反向；全注意力前向通过全局log-sum-exp合并（CP8 MAX+SUM all-reduce）实现精确分区软max，但反向中dQ全规约，dK/dV保持本地，导致K/V适配器梯度不同步。
- **针对GLM-5.2**（MLA/DSA压缩注意力，MoE FFN）：
  - 提示状态：78层的MLA潜变量页 + 21个索引计算层的DSA索引键页，全部存放在CPU内存中（共约5.8 GiB/rank，32 rank总共186 GiB）。每层重放时仅将当前层所需页暂存到GPU，执行后释放。
  - 响应重放：使用完整层级别的激活检查点（checkpointing），保存短层输入，反向时重新计算整个层（包括注意力、路由、MoE前馈）。DSA索引在接收到的证明中仅基于本地CP分片（65k tokens）做top-2048选择，未进行跨rank全局选择。
- **优化器作用域**：两个模型均采用LoRA/QLoRA适配器，优化器仅更新低秩参数；基权重、嵌入等冻结。Qwen使用NF4 QLoRA，GLM使用rank-8 LoRA。

### 公式与算法流程（文字说明）
- GRPO目标函数（公式2）基于组内相对优势，结合截断重要性比率和KL惩罚。LongStraw关注的是如何高效计算其条件对数概率，而不是修改目标。
- 更新图分为5个阶段：提示捕获（无梯度）→ 冻结评分 → 优势构建 → 策略重放（逐成员）→ 优化器事务。
- 由于提示状态被stop-gradient，损失关于θ的梯度仅包含显式响应贡献，缺失通过提示状态的间接梯度项（公式3的第二项）。因此执行结果不等同于完整序列梯度等价。

## 6. 论文的主要结论与发现
1. **固定预算下的长上下文GRPO执行是可行的**：在8块H20上实现2.1M位置（原始设置8倍），在32块H20上实现2.1M位置（原始设置2倍），且组大小增加对峰值内存影响极小（+0.21GB）。
2. **状态生命周期和物理所有权是决定上下文限制的关键**：逻辑分片不等同于物理释放；必须将页复制为独立紧凑张量才能有效释放内存。
3. **前向保真度、更新一致性和梯度等价性是完全不同的概念**：论文明确划分了四个证据水平（执行容量、响应运算符保真度、分布式更新一致性、全梯度等价性），并承认当前结果仅达到前两个级别。
4. **稠密和MoE模型将峰值压力转移到不同位置**：Qwen压力集中在全注意力的KV页；GLM压力集中在索引工作空间和MoE路由后的扩展中间张量。
5. **上下文并行和专家并行是正交不不可互换的**：CP处理token历史分配，EP处理专家参数分配，其通信语义不同。
6. **4.25M上下文可在8块H20上执行**（但要求显式的prefix-frozen参数化，即提示位置的适配器梯度被冻结），并完成了8个优化器步骤（64次成员重放）。

## 7. 优点
- **新颖的问题视角**：将长上下文训练的内存问题重新定义为**张量生命周期和所有权问题**，而非单纯优化单个算子或增加设备。
- **架构通用的执行栈**：在两个差异很大的模型（稠密混合注意、MoE压缩注意）上均成功实现，且核心设计原则（一次捕获、串行重放）一致。
- **严谨的证据层次**：明确区分“执行容量”、“响应运算符保真度”、“分布式更新一致性”、“全梯度等价性”，并在每个receipt中声明当前所处的级别，不夸大claim。
- **详细的系统分析**：揭示了Qwen缺失的K/V适配器梯度规约、GLM的CP局部DSA选择和跳过的gradient finalization等具体缺口，为后续修复提供路线图。
- **开源代码**（GitHub链接），促进可复现研究。
- **资源效率**：在固定且相对经济的GPU数量（8或32块H20）下实现百万token级后训练，降低了研究门槛。

## 8. 不足与局限
- **提示状态被分离（stop-gradient）**：响应反向传播未恢复通过提示状态的梯度项（公式3的第二项），因此不是完整的序列梯度训练。
- **分布式梯度组成不完整**：
  - Qwen：dK/dV适配器梯度未跨CP rank规约，导致8个AdamW实例从不同的本地梯度更新，参数可能发散。
  - GLM：自定义重放路径跳过了Megatron的gradient finalization，CP复制的注意力/稠密FFN/输出头适配器梯度未被规约，而分发优化器假设已规约。
- **前向运算符保真度不足（GLM）**：DSA的top-2048选择仅基于每个rank本地的65k token shard，而非全局2.1M token，不是模型定义的全上下文运算符。
- **工作负载局限**：
  - 使用合成token和确定性奖励，无在线采样、环境交互、奖励模型。
  - Qwen使用β=0且old/reference来自同一模型，导致重要性比为1，截断分支未激活；GLM虽然设置β=0.01但first-step ratio=1，KL为零。
  - 未验证策略改进或下游任务性能。
- **实验充分性不足**：
  - 仅单次运行，无方差。
  - 未进行组大小扫描（仅G=2和G=8两个点）。
  - 未测试GLM在更高上下文（>2.1M）的能力上限。
  - 未报告完整资源账目（主机内存、网络流量、能耗）。
- **可重现性缺口**：最终GLMreceipt未绑定学习率、token校验和、LoRA种子、软件版本等关键元数据。
