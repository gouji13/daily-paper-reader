## 摘要
自我进化框架通常优化任务解决方案，同时将周围的框架视为固定不变。我们提出了感知框架的自我进化（HASE），这是一个基于智能体的强化学习框架，其中单个模型可以在多轮动作空间中生成任务解决方案或编辑选定的框架组件。HASE使单个Qwen3-8B模型能够匹配使用Claude Code作为框架提议者的GPT-OSS-120B模型的文本分类性能。在alpha因子挖掘中，HASE超越了已报告的GPT-OSS-120B基线。HASE还能修复不完美的评估组件，并在圆填充算法发现中收敛到最先进的性能。这些结果表明，HASE通过一个统一的智能体过程改进了框架和解决方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **背景**：当前自进化系统（如 FunSearch、AlphaEvolve、OpenEvolve 等）通常将模型周围的“harness”（包括提示词、记忆、检索策略、评估器等）视为固定不变，仅优化任务层面的解决方案（如代码或算法）。这种做法忽略了 harness 对模型性能的决定性影响。
- **问题**：固定 harness 存在两大局限性：
  1. 人工编写的评估器可能不完整或与真实环境不一致，导致模型利用评估漏洞进行“奖励冲浪”（reward hacking）。
  2. 即使评估器正确，指导性 harness（如提示、记忆、检索策略）可能提供劣质的搜索先验，限制模型发现更好解。
- **核心含义**：需要让模型在进化过程中也能改进其自身的工作环境（harness），实现模型权重、任务解决方案和 harness 的协同进化，从而突破固定 harness 的性能上限。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：提出 Harness-Aware Self-Evolving (HASE) 框架，将任务解决方案生成和 harness 编辑置于同一个智能体强化学习的多轮动作空间中。同一模型可以选择提交解决方案或修改选定的 harness 组件。
- **Harness 分类**：
  - **指导性组件（Guidance Components）**：包括提示词、记忆格式化、上下文检索脚本等，用于改进搜索质量，可自由编辑。
  - **评估性组件（Evaluation Components）**：包括局部验证器和评分器，用于定义解的合法性和指标。编辑这类组件需要基于真实世界评估器的反馈锚定，防止无限制地篡改奖励。
- **关键技术细节**：
  - **统一动作空间**：模型可执行 `ls`、`cat`、`grep`、`replace`、`overwrite` 等沙盒命令来检查和修改 editable harness，也可直接提交解决方案。
  - **奖励设计**：仅对提交解决方案的轨迹给予奖励，不奖励 harness 操作本身。对于评估性编辑，奖励与局部评估器与真实评估器在“不一致集合”上的匹配度成正比。
  - **阶段级审查（Phase-level Review）**：每个阶段内运行多个 GRPO 训练 epoch；阶段结束时，收集候选 harness 编辑，过滤、排序后仅提交能提升验证性能（指导性组件）或修复不一致集合（评估性组件）的编辑，并写入持久 harness。
- **算法流程**（伪代码描述）：
  1. 初始化策略 πθ 和持久 harness h。
  2. 若允许评估性进化，初始化不一致集合 M。
  3. 迭代多个阶段，每个阶段内：
     - 进行多个 GRPO epoch，模型生成多轮轨迹（可选编辑 harness 或提交解），使用局部评估器评分，更新策略。
     - 阶段结束时，若允许评估性进化，用真实评估器检查并收集不一致案例到 M。
     - 根据规则审查候选编辑，提交最佳更新到持久 harness。

## 3. 实验设计：数据集/场景、基准、对比方法
