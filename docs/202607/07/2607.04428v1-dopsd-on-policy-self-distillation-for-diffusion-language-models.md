## 摘要
扩散大语言模型通过迭代去噪掩码序列来生成文本，为自回归模型提供了一种并行替代方案，但通过后训练引出强推理能力仍然困难：监督微调是离策略的，存在曝光偏差；而强化学习仅提供稀疏的序列级奖励，且在没有可处理序列似然的情况下难以应用。在线自我蒸馏（OPSD）提供了一种有前景的替代方案，将同一个模型同时用作学生和教师，提供密集的、词元级别的在线监督，但其有效性依赖于给予教师特权信息——通常是推理时无法获得的实例特定真实参考——最终导致学生蒸馏出一个弱无特权共识策略，对dLLM推理改进甚微。我们提出dOPSD，它直接从学生自身的去噪轨迹中推导教师的特权，利用同一轨迹中后期更确定步骤来评估掩码位置，而非外部标签；因此教师的优势来自模型自身的解码过程。在Dream和LLaDA上，dOPSD同时提升了领域内数学推理和领域外代码生成，优于监督和在线策略基线。

## Abstract
Diffusion large language models (dLLMs) generate text by iteratively denoising a masked sequence, offering a parallel alternative to autoregressive models, but eliciting strong reasoning through post-training remains difficult: supervised fine-tuning is off-policy and suffers from exposure bias, while reinforcement learning gives only sparse, sequence-level rewards and is hard to apply without tractable sequence likelihoods. On-policy self-distillation (OPSD) offers a promising alternative, using one model as both student and teacher to provide dense, token-level, on-policy supervision, but its effectiveness hinges on giving the teacher privileged information (PI) - typically an instance-specific ground-truth reference unavailable at inference - so the student ends up distilling a weak PI-free consensus policy that yields little improvement on dLLM reasoning. We introduce dOPSD, which instead derives the teacher's privilege directly from the student's own denoising trajectory, evaluating masked positions using later, more-decoded steps of that same trajectory rather than an external label, so the teacher's advantage emerges from the model's own decoding process; on Dream and LLaDA, dOPSD improves both in-domain math reasoning and out-of-domain code generation, outperforming supervised and on-policy baselines.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题背景**：扩散语言模型（dLLM）通过迭代去噪掩码序列并行生成文本，是自回归模型的竞争方案。但后训练阶段提升其推理能力面临两大障碍：
  - **监督微调（SFT）** 是离策略的，存在曝光偏差，训练和推理时的前缀分布不一致。
  - **强化学习（RL）** 仅提供稀疏的序列级奖励，且扩散模型缺乏可处理的序列似然，RL方法难以适配。
- **现有在线自蒸馏（OPSD）的困境**：OPSD利用同一模型同时作为学生和教师，通过教师拥有特权信息（PI）提供密集、在线、token级的监督。但PI通常来自外部实例特定的真实参考（如完整参考解），推理时不可用。理论分析表明，这种设置导致学生只能收敛到一个弱化的“无特权共识”策略，对数学推理这类实例特定场景几乎没有提升。
- **本文目标**：提出dOPSD，将教师的特权信息从外部标签迁移到模型**自身的去噪轨迹**中，利用解码过程中后期更确定的步骤来监督早期步骤，既保留密集在线信号，又摆脱对外部参考解的依赖。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：在扩散模型解码轨迹中，**后期状态**（已完成更多token）相比**早期状态**拥有更多的上下文信息。将早期状态作为学生，后期状态作为教师，教师对学生仍处于掩码位置的预测进行蒸馏引导，无需任何外部答案或参考链。
- **关键技术细节**：
  - **学生状态（Student state）**：随机采样解码步骤 \(k\)（满足掩码比例 \(> \tau\)，实验中 \(\tau=0.5\)），学生评估该步骤下真正掩码位置 \(M_k\) 的预测分布 \(\pi_\theta(\cdot|x, s_k)_i\)。
  - **教师目标（Teacher target）**：对于掩码位置 \(i\)，计算**未来所有继续处于掩码状态**的步骤 \(\{t: k < t \le K, i \in M_t\}\) 上的预测分布的平均值 \(\bar{p}_i\)。若下一句就解码（\(T_i=\emptyset\)），则退回到最终补全 \(\hat{y}\) 的预测。教师梯度被截断（stop-gradient）。
  - **蒸馏损失**：使用 token 级的广义 Jensen-Shannon 散度（默认取前向KL \(\beta \to 0\)），仅对掩码位置 \(M_k\) 计算：
    \[
    \ell(x, \hat{y}) = \frac{1}{|M_k|} \sum_{i\in M_k} \text{JSD}_\beta( \text{sg}[\bar{p}_i] \parallel \pi_\theta(\cdot|x, s_k)_i )
    \]
  - **Rollout验证**：只有最终答案正确（通过短答案验证）的 rollouts 才参与蒸馏；错误 rollout 梯度为0。该验证是可选的（无验证依然有效）。
- **算法流程（文字版）**：
  1. 对每个prompt \(x\)，模型采样得到一个完成 \(\hat{y}\) 并记录解码轨迹 \((s_0,\dots,s_K)\)。
  2. 若 \(\hat{y}\) 答案正确，则随机选取一个中间步骤 \(k\)（掩码比例 > \(\tau\)）。
  3. 对 \(k\) 步的每个掩码位置，从后续轨迹中收集仍掩码的步骤，计算教师平均分布 \(\bar{p}_i\)。
  4. 用 token 级 JSD 散度对学生分布进行蒸馏，梯度只更新学生部分。

### 6. 论文的主要结论与发现

- **dOPSD是唯一全面改善base模型的方法**：在Dream上GSM8K提升1.6点，MATH500提升3.2点；在LLaDA上MATH500提升4.8点。同时领域外代码生成（HumanEval/MBPP）也获得显著提升，证明提炼的推理能力泛化到编程。
- **SFT、GRPO、OPSD所有基线均未持续优于base**：SFT因暴露偏差下降；GRPO受限于稀疏奖励；OPSD (answer-only) 几乎没有增益；OPSD (full-solution) 严重崩溃（数学推理降低10+点，代码生成也暴跌）。
- **失败原因验证**：OPSD崩恰好符合PI驱动的理论预测——实例特定的PI导致弱共识。dOPSD通过从自身轨迹获取特权，彻底避免了这一问题。
- **消融确认**：前向KL优于反向KL；全教师视野最好；中等掩码阈值（τ=0.5）最优；rollout验证可带来额外增益但不是必须。

- **方法论亮点**：

### 8. 不足与局限

- **模型规模限制**：仅实验了7B和8B参数级模型，更大规模（如70B）尚未验证，教师视野的计算成本随模型规模和序列长度线性增加，扩展到更大模型可能需要近似或截断策略。
- **计算开销**：构建教师目标需要对每个中间步骤的**未来所有掩码步骤**做前向预测，相当于在rollout后额外执行多次前向（每次对应一个未来步骤）。虽然文中声称“只增加一次教师前向”，但实际上对每个位置要遍历其后仍掩码步骤，对于长序列和多步解码，开销不可忽视。论文未给出具体训练时间对比。
- **评测任务范围有限**：仅包含数学推理和代码生成，未测试其他推理任务（如常识推理、科学问答）或开放式生成任务。代码生成虽然属于OOD，但仍是封闭域测试（单元测试验证）。dOPSD对生成多样性、创意类任务的影响未知。
- **基线覆盖不足**：虽对比了GRPO（RL方法），但未与更先进的RL变体（如DPO on diffusion、或基于过程奖励的RL）比较。OPSD作为直接移植已是合理对比，但缺少知识蒸馏领域其它方法（如在线蒸馏、反向KL蒸馏等）。
- **LoRA vs 全参数微调**：所有方法均使用LoRA（rank=32），虽确保了公平对比，但可能限制dOPSD的最终潜力。全参数微调下dOPSD是否带来更大增益未探究。
- **回答验证的依赖**：主实验中需要最终答案进行rollout筛选，虽然消融证明无验证仍有效，但这一“验证”在无答案场景下需设计替代方案（如自洽性检查）。
