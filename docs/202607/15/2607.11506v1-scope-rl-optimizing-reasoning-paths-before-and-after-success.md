## 摘要
基于可验证奖励的强化学习（RLVR）利用稀疏的可验证最终答案奖励来优化大语言模型。这种稀疏锚点能够可靠地验证轨迹是否成功，但无法对产生该轨迹的推理路径提供直接反馈。在成功之前，解决难题的初步进展无法获得奖励信号；在成功之后，结果奖励无法区分组织良好的正确轨迹与冗余或局部有缺陷的轨迹。我们提出了SCOPE-RL（带过程效率的支架式链优化），这是一个两阶段框架，它在保留GRPO更新的同时强化这一锚点：自适应支架式强化学习在成功前的隐藏答案子问题链上添加前缀分解的可验证奖励；质量感知过程强化学习在成功之后应用正确性门控的过程形状奖励来优化正确轨迹。一个经专家验证的步骤质量评估协议评估有用步骤密度、错误定位和令牌效率，超越最终答案准确性。在使用DAPO-Math和Big-Math训练的Qwen3-8B-Instruct上，SCOPE-RL相比仅使用最终输出的GRPO将平均准确率提高了多达11.2个百分点，并将推理令牌减少了多达27.1%；这些优势在GSPO下以及在Qwen3-0.6B-Instruct上得以保持，表明奖励信号强化与策略更新层面的RLVR进展是互补的。代码和数据可在https://github.com/tokencraft-lab/SCOPE-RL获取。

## Abstract
Reinforcement learning with verifiable rewards (RLVR) optimizes LLMs using sparse verifiable final-answer rewards. This sparse anchor reliably verifies whether a trajectory succeeds but provides no direct feedback on the reasoning path that produced it. Before success, prerequisite progress on hard problems receives no reward signal; after success, outcome rewards cannot distinguish well-organized correct trajectories from redundant or locally flawed ones. We introduce SCOPE-RL (Scaffolded Chain Optimization with Process Efficiency), a two-stage framework that densifies this anchor while retaining the GRPO update: Adaptive Scaffolded RL adds prefix-decomposed verifiable rewards on answer-hidden sub-question chains before success, and Quality-Aware Process RL applies correctness-gated process-shape rewards to refine correct trajectories after success. An expert-validated Step-Quality Evaluation Protocol evaluates useful-step density, error localization, and token efficiency beyond final-answer accuracy. On Qwen3-8B-Instruct trained on DAPO-Math and Big-Math, SCOPE-RL improves average accuracy by up to 11.2 pp and reduces reasoning tokens by up to 27.1% over outcome-only GRPO; the gains hold under GSPO and on Qwen3-0.6B-Instruct, indicating that reward-signal densification is complementary to policy-update-level RLVR advances. Code and data are available at https://github.com/tokencraft-lab/SCOPE-RL.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：基于可验证奖励的强化学习（RLVR）仅对最终答案的正确性给出稀疏奖励（0/1），这在成功前和成功后都造成信息缺失：
  - **成功前**：对难题的局部正确进展（如中间子问题正确）不给予任何奖励信号，导致训练停滞。
  - **成功后**：无法区分正确回答中的高质量推理与冗余、含局部错误的推理，模型会倾向于产生冗长但低效的轨迹。
- **研究动机**：作者将这种“稀疏锚点”问题统一表述，并提出通过奖励信号密集化来补全信息，而不改变底层策略更新（GRPO）本身，使两阶段方法（成功前和成功后）互补，提升推理质量和效率。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：在保留GRPO更新的前提下，分别在成功前和成功后引入不同的密集奖励信号，且两阶段自然顺序执行。
- **阶段1：自适应支架式强化学习（ASR）**  
  - **针对问题**：成功前，多数轨迹无法到达最终答案，端点奖励缺乏信号。
  - **关键步骤**：
    1. **答案隐藏的支架构造**：离线为每个训练问题生成一组子问题链（sub-question chain），每个子问题有独立可验证答案，但子答案不对策略暴露（用于奖励计算）。
    2. **按策略路由**：在训练时，对每个问题用原始提示采样G条轨迹，计算平均结果奖励；若低于阈值τ（实验中τ=0.5），则改用支架提示重新采样，并应用支架奖励。
    3. **前缀一致性支架奖励**：  
       - 定义前缀一致性指标Π_i = ∏_{j=1}^i 1[ˆa_j = a_j]（前i个子问题全部正确才为1）。  
       - 奖励公式：R_ASR = β·(1/m)∑_{i=1}^m Π_i + (1-β)·Π_m·1[ˆa_main = a]，其中β=0.5。  
       - 该奖励只在所有子问题前提正确时才给最终主答案奖励，从而鼓励逐步进步而非跳步猜测。
- **阶段2：质量感知过程强化学习（QPR）**  
  - **针对问题**：成功后，等式奖励无法区分推理质量。
  - **关键步骤**：
    1. **正确性门控**：仅对最终答案正确的轨迹应用过程奖励；错误轨迹奖励置0。
    2. **步骤级类别标注**：使用固定LLM Judge（GPT-4.1-mini）将正确轨迹的推理步骤标注为五类：useful, mechanical, redundant, reversion, error。
    3. **质量感知奖励组合**：  
       - 有用步骤比例：S_u = (1/N)∑ 1[c_i = useful]。  
       - 低价值惩罚因子：Φ = ∏_{k∈K} (1-λ_k r_k)，r_k为k类步骤比例，λ_k分别为0.05,0.20,0.25,0.40。  
       - 简明激励项：κ = 1/(1+α ln(N+1))，α=0.5。  
       - 最终过程奖励：Q_process = S_u · Φ · κ。  
    4. 阶段2在原始未分解问题上训练，使模型内化过程偏好。
- **两阶段训练流程**：阶段1产生中间检查点，阶段2从该检查点初始化继续训练。混合奖励会削弱门控效果，故不混合。

## 7. 优点：方法或实验设计上的亮点

- **统一框架**：将“成功前”和“成功后”的稀疏奖励问题统一为“稀疏锚点”，提出互补的密集化策略。
- **无答案泄漏**：ASR的支架子问题答案对策略隐藏，仅用于奖励，保持RLVR的可验证性。
- **正确性门控**：QPR只对正确轨迹施加过程奖励，避免鼓励短而错的回答。
- **设计干净**：两阶段自然顺序，不修改GRPO更新本身，易于集成到现有系统中。
- **充分验证**：在多个基准、两种训练数据、不同模型大小、替代优化器（GSPO）、人体偏好评估等多个维度上进行了全面评估，实验严谨。
- **步骤质量评估协议**：提供了一个标准化的推理过程评估方法，且验证了与人类专家的一致性。
