## 摘要
强化学习（RL）在增强大语言模型（LLMs）的推理能力方面取得了显著成功。然而，广泛使用的无评论家RL方法依赖于均匀信用分配，即无论标记之间的差异如何，都将相同的优势广播给所有标记。我们发现了这种设计的一个关键失败模式，称之为正信用污染：低概率的尾部标记（上下文错误）与同一轨迹中的合理标记获得相同的正信用，导致错误推理行为被不加区分地强化。为解决此问题，我们提出了尾部感知信用校准（TACO），一种校准均匀信用分配以抑制不良正更新的方法。TACO首先计算一个尾部风险得分，该得分结合局部生成上下文来评估每个标记落入不可靠尾部的风险，从而区分意外罕见性与不确定性驱动的探索。然后TACO使用该得分调整高风险标记的正信用，而不完全移除它们的梯度，从而使得重复出现的有用罕见模式能够累积强化，同时间歇性噪声逐渐被抑制。在三个LLM和八个基准上的实验结果表明，TACO持续优于GRPO风格的基线。值得注意的是，TACO提高了训练稳定性，支持在长程RL中持续的性能提升。源代码可在https://github.com/xiuyilou/TACO获取。

## Abstract
Reinforcement learning (RL) has achieved remarkable success in enhancing the reasoning capabilities of large language models (LLMs). However, widely used critic-free RL methods rely on uniform credit assignment, broadcasting the same advantage to all tokens regardless of their differences. We identify a critical failure mode of this design, which we refer to as Positive-Credit Contamination: low-probability tail tokens that are contextually erroneous receive identical positive credit to plausible ones within the same trajectory, resulting in the indiscriminate reinforcement of flawed reasoning behavior. To mitigate this issue, we propose Tail-Aware Credit calibratiOn (TACO), a method that calibrates uniform credit assignment to suppress undesirable positive updates. TACO first computes a tail-risk score that incorporates the local generation context to assess each token's risk of falling into the unreliable tail, distinguishing unexpected rarity from uncertainty-driven exploration. TACO then uses this score to tune positive credit for risky tokens without removing their gradients entirely, so that recurring useful rare patterns can accumulate reinforcement while incidental noise is progressively dampened. Experimental results across three LLMs and eight benchmarks show that TACO consistently outperforms GRPO-style baselines. Notably, TACO improves training stability, supporting sustained performance gains in long-horizon RL. The source code is available at: https://github.com/xiuyilou/TACO.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：在基于可验证奖励的强化学习（RLVR）中，广泛使用的无评论家方法（如 GRPO）采用**统一信用分配**，将相同的序列级优势广播给所有生成的 token。这种设计导致一个关键失败模式——**正信用污染（Positive-Credit Contamination）**：上下文中语义错误的低概率“尾部 token”（implausible tail tokens）与合理的 token 获得相同的正优势，从而被不加区分地强化，逐渐使策略偏向有缺陷的推理模式。
- **背景**：尽管 GRPO 在推理任务上表现优异，但其忽略了 token 在不同上下文下的可靠性差异。以往方法依赖外部信号或内在不确定性（如熵）来分配信用，但缺乏局部语义视角，无法识别尾部 token 的不可靠性，甚至可能放大它们的不良影响。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：通过局部生成上下文评估每个 token 的“尾部风险”，然后对高风险 token 的正信用进行**软衰减**，保留有利于探索的罕见模式，同时抑制噪声。
- **关键技术细节**：
  1. **自适应尾部风险估计**：结合 token 采样概率 $p_{i,t}$ 和局部熵 $H_{i,t}$，定义风险分数 $r_{i,t}^{\text{tail}} = -\log p_{i,t} - H_{i,t} + \log \alpha$。当分数大于 0 时，认为该 token 属于不可靠尾部。参数 $\alpha$ 控制风险识别的激进程度。
  2. **尾部感知信用校准**：根据风险分数计算权重 $w_{i,t}$：
     - 若 $r_{i,t}^{\text{tail}} > 0$，则 $w_{i,t} = 1 - \lambda \Big(1 - \exp(-r_{i,t}^{\text{tail}})\Big)$，其中 $\lambda \in (0,1)$ 控制最大抑制强度；
     - 否则 $w_{i,t} = 1$。
  3. **校正后的优势**：仅对正优势进行调制，负优势保持不变。最终 token 级优势为 $\hat{A}_{i,t}^{\text{TACO}} = w_{i,t}^{\mathbb{I}[\hat{A}_i > 0]} \hat{A}_i$。
  4. **集成到 GRPO**：用上述校准后的优势替换原始广播优势，其他组件不变。
- **算法流程**：
  1. 采样组完成 → 计算奖励和组归一化优势；
  2. 根据局部概率和熵计算每个 token 的尾部风险分数；
  3. 将风险分数转换为权重，并校准正优势；
  4. 使用校准后的优势进行标准 GRPO 策略更新。

## 7. 优点
- **轻量高效**：仅利用前向传递中已有的 token 概率和局部熵，无需额外推理或辅助模型，计算开销可忽略。

- **稳定性与泛化性**：自动适应不同模型的不确定性分布，无需模型特定调参；支持多步长程优化而不会性能退化。
- **实践友好**：直接集成到 GRPO 训练循环中，算法简单，源码公开。
