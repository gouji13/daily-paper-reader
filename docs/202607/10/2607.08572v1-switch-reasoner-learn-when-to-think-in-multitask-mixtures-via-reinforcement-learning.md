## 摘要
多模态大语言模型（MLLMs）通常遵循固定的“思考-然后-回答”范式，这在异构多任务场景中效率低下，因为简单输入可能不需要显式推理，而困难输入则能从中显著受益。在后训练过程中，学会何时思考也不稳定，因为不平衡的展开可能导致模型倾向于始终思考或始终直接回答。我们提出Switch-Reasoner，一种基于GRPO的框架，学习为MLLMs自适应选择推理模式。它将思考视为虚拟工具调用，允许模型直接回答或在回答前调用显式推理。为了稳定这一决策，我们引入了一种双层调节机制，平衡思考模式和直接模式的整体使用，同时基于两种选择的相对收益提供样本级监督。在11项多模态任务上的实验表明，Switch-Reasoner在减少不必要推理的同时保持了强性能，实现了更好的准确性与效率权衡。

## Abstract
Multimodal Large Language Models (MLLMs) often follow a fixed Think-then-Answer paradigm, which is inefficient in heterogeneous multitask settings because simple inputs may not require explicit reasoning while difficult ones can benefit substantially from it. Learning when to think is also unstable during post-training, where imbalanced rollouts can drive the model toward always-thinking or always-direct behavior. We propose Switch-Reasoner, a GRPO-based framework that learns to adaptively select reasoning modes for MLLMs. It treats thinking as a virtual tool invocation and allows the model to either answer directly or invoke explicit reasoning before answering. To stabilize this decision, we introduce a dual-level regulation mechanism that balances the overall use of Thinking Mode and Direct Mode while providing sample-level supervision based on the relative benefit of the two choices. Experiments on 11 multimodal tasks show that Switch-Reasoner reduces unnecessary reasoning while maintaining strong performance, achieving a better accuracy-efficiency trade-off.

## 2. 方法论

### 核心思想
- **Thinking-as-Tool**：将显式推理（Thinking Mode）重新定义为虚拟工具调用，而非隐式的始终开启的生成模式。模型通过可切换的提示（switchable prompt）先决定是直接输出答案（Direct Mode）还是调用思考工具（Thinking Mode），从而将“是否思考”转化为一个可观测、可优化的离散路由动作。

### 关键技术细节
1. **可切换提示（Switchable Prompt）**：模型首先生成包含两种可能的响应：要么输出 `<answer>...</answer>` 直接回答，要么输出 `<tool_call>{"name":"require_think","arguments":{}}</tool_call>` 调用思考工具。若调用工具，后续的“think-tool response”提示会引导模型进行内部思考并给出答案。模式决策被形式化为策略分解：πθ(o, m|q) = πθ(m|q)πθ(o|q, m)，其中 m∈{0,1} 表示是否思考。
2. **全局模式平衡控制（Global Mode Balance Control）**：一个在线延迟控制器，通过上一批次的统计量（各模式的使用比例和准确率差异）生成全局控制信号 B，对当前批次中暂时不受欢迎的模式进行乘性惩罚（衰减因子 η），防止策略崩塌到单一模式。同时包含极端不均衡保护（当某模式使用率过高时强制引入相反信号）。
3. **样本级细粒度优化（Sample-level Fine-grained Optimization）**：为每个训练样本分别固定评估两种模式（直接回答和思考）的成功率，计算思考相对于直接回答的增益 Δi，然后划分三类：must-think（思考带来明显提升）、safe-direct（直接回答已可靠且思考无增益）、uncertain（不确定）。据此给出对应的路由奖励 r_route，鼓励匹配偏好模式。
4. **奖励计算**：最终奖励 r* = r_global + r_route，其中 r_global 由基础奖励（准确率+格式合规）经全局控制信号调整得到，r_route 根据样本偏好和实际选择给出正负奖励。然后用于 GRPO 的组内优势估计。
