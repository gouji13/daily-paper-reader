## 摘要
传统的语言模型蒸馏通常依赖于固定的教师生成数据，这可能无法涵盖不断进化的学生策略所遇到的状态。在线策略蒸馏（OPD）则收集教师或评估者在学生生成的轨迹上的监督信号。然而，现有的OPD方法在监督形式、分词器兼容性、教师访问方式和监督粒度上存在显著差异，导致实现碎片化，难以复现和扩展。我们提出 EasyOPD，一个基于 verl（一个用于大型语言模型的分布式强化学习框架）构建的在线策略蒸馏框架。EasyOPD 分离了用户侧配置、方法特定的监督逻辑以及基于 verl 的执行。其方法模块通过用于损失构建、轨迹元数据、奖励处理、分词器对齐和教师侧计算的扩展边界连接到共享后端。我们针对三种OPD设置——跨分词器OPD、在线自蒸馏和逐步骤OPD——实例化了代表性方法。在推理、代码生成、科学知识和工具使用基准上的实验表明，这些实现可以通过相同的基于 verl 的后端执行，同时保留其方法特定的目标和任务相关的性能特征。我们发布了 EasyOPD，包含可运行的 YAML 配置、文档、可安装的演示包和视频。

## Abstract
Conventional language-model distillation often relies on fixed teacher-generated data, which may not cover the states encountered by an evolving student policy. On-policy distillation (OPD) instead collects teacher or evaluator supervision on student-generated rollouts. However, existing OPD methods differ substantially in supervision form, tokenizer compatibility, teacher access, and supervision granularity, leading to fragmented implementations that are difficult to reproduce and extend. We present \textsc{EasyOPD}, an on-policy distillation framework built on verl, a distributed reinforcement-learning framework for large language models. \textsc{EasyOPD} separates user-side configuration, method-specific supervision logic, and verl-based execution. Its method modules connect to the shared backend through extension boundaries for loss construction, rollout metadata, reward processing, tokenizer alignment, and teacher-side computation. We instantiate representative methods for three OPD settings -- cross-tokenizer OPD, on-policy self-distillation, and step-wise OPD. Experiments on reasoning, code-generation, scientific-knowledge, and tool-use benchmarks show that these implementations can be executed through the same verl-based backend while retaining their method-specific objectives and task-dependent performance profiles. We release \textsc{EasyOPD} with runnable YAML configurations, documentation, and an installable demonstration package and video.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：传统知识蒸馏（Knowledge Distillation, KD）依赖教师模型在固定上下文上生成的响应或 token 分布进行离线训练。然而，学生模型在自回归推理时会基于自身的前缀进行生成，可能遇到离线监督未覆盖的状态，导致训练-推理分布不匹配，错误逐步累积。
- **现有OPD的碎片化**：在线策略蒸馏（On-Policy Distillation, OPD）通过收集当前学生生成轨迹上的教师监督来缓解此问题，但不同OPD方法在监督形式（logit、rubric、step-wise）、分词器兼容性、教师访问方式、监督粒度等方面差异巨大，导致实现碎片化，难以复现和扩展。
- **动机**：现有框架（如TRL、verl、slime、KDFlow）各覆盖部分功能，但无一提供统一的、带有明确扩展边界的轻量级OPD抽象，使得用户在切换方法或添加新方法时需要修改大量底层代码。
- **贡献**：提出 **EasyOPD**，一个基于 verl 构建的三层架构框架，分离用户配置、方法级监督逻辑和分布式执行，通过薄钩子（thin hooks）连接方法模块到共享后端，支持跨分词器OPD、在线自蒸馏、逐步骤OPD等多种设置，且可通过YAML配置切换方法，大幅降低复现和扩展门槛。

## 2. 方法论

### 核心思想
EasyOPD 将一次 OPD 训练分解为四个可复用阶段：方法调用、配置解析、监督构建和分布式执行。通过三个分离的层实现统一：

- **用户层**：提供 Python/CLI 入口点和 YAML 配置文件（方法、模型、数据集、超参数），用户只需选择方法并更改 YAML 即可切换。
- **EasyOPD层**：管理方法注册、方法级模块和诊断日志，通过钩子（hooks）与 verl 交互。钩子包括损失（loss）、轨迹元数据（rollout metadata）、奖励（reward）、对齐（alignment）和教师侧边车（teacher sidecar）等。
- **verl执行层**：复用 verl 的分布式 rollout、模型执行、奖励计算、优化等基础设施。

### 关键技术细节（钩子与扩展点）
- **损失钩子**：用于 token 级或分布级监督。
- **轨迹元数据钩子**：用于附加上下文或步骤注释。
- **奖励钩子**：用于法官或评分反馈。
- **对齐钩子**：用于跨分词器的 token 或跨度映射。
- **教师侧边车钩子**：用于教师侧信号（logits、隐藏状态、法官输出等）。

方法模块只需注册所需的钩子，即可在共享的 verl 流水线上插入特定的监督逻辑，而不需要修改 verl 核心代码。例如，基于 logit 的 GKD 只注册一个损失钩子；跨分词器 SimCT 注册对齐和教师侧边车钩子；逐步骤 SOD 注册损失和轨迹元数据钩子。

### 算法流程（文字描述）
1. 学生策略根据当前策略生成 on-policy 轨迹。
2. 根据所选方法，在轨迹上构造监督信号（如教师 logits、对齐后的 token 分布、法官奖励或步骤级反馈）。
3. 通过钩子将监督信号传递给训练系统，用于损失计算、奖励聚合或轨迹级加权。
4. 使用 verl 的分布式训练引擎更新学生模型。
5. 迭代重复，每一步都基于更新后的学生策略生成新轨迹。

## 6. 主要结论与发现

- EasyOPD 成功地将跨分词器OPD、在线自蒸馏和逐步骤OPD三种设置统一在同一基于 verl 的后端上，仅通过 YAML 配置切换方法，而无需修改底层代码。
- 三种设置下的实现均保留了原方法的性能特征：SimCT在跨分词器设置下取得最佳平均分（53.5），SDPO在在线自蒸馏中化学与生物提升显著，SOD在逐步骤设置下全面超过GRPO和Response-level OPD（平均39.95 vs 34.81）。
- 结论：EasyOPD 通过薄钩子设计，实现了方法级监督逻辑与分布式执行的分离，降低了 OPD 方法复现和扩展的难度。

## 7. 优点

1. **统一性**：首次提出了一个涵盖三种不同监督接口的 OPD 框架，并通过钩子抽象实现了高扩展性。
2. **易用性**：用户只需修改 YAML 文件即可切换方法，无需修改训练脚本；框架提供诊断指标（如重叠词表大小、有效跨度比例等），便于调试。
3. **代码开源**：附带完整 YAML 配置、文档、可安装演示包和视频，复现门槛低。
4. **继承 verl 优势**：直接复用 verl 的分布式 rollout、模型并行、张量并行、vLLM 推理等成熟基础设施，无需重新实现。
5. **方法模块化**：开发者只需注册钩子即可添加新方法，且不影响现有流程，便于社区贡献。
