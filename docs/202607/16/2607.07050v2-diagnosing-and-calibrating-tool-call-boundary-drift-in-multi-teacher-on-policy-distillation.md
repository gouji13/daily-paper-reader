## 摘要
自主语言模型必须学会何时调用工具、何时使用工具响应以及何时直接回答。这使得多教师同策蒸馏成为一种自然的训练策略：一个教师可专精于工具调用，另一个专精于直接回答，而学生可从两者在其自身生成分布上的输出中学习。我们表明，该策略可能引发一种行为偏移，而仅凭聚合损失无法察觉。在双教师工具使用场景中，标准广义知识蒸馏提升了工具调用召回率，但也导致模型过度调用，即在应直接回答的样本上也调用工具。聚合解释并不充分：工具调用样本并未获得更多token暴露，且工具调用教师的完整序列每个token的散度也并非更大。我们转而分析行为杠杆不平衡：在模式入口和结构位置（如<tool_call>和函数名）处的局部token级信号可能对全局生成模式具有不成比例的控制力。我们提出Soft Clamp，一种逐token散度校准方法，它动态压缩极端的token级Jensen-Shannon散度，同时保留非零梯度。在APIGen-MT上，与标准GKD相比，Soft Clamp将过度调用从13.7%降至9.0%，同时保持决策准确率相当。在BFCL多轮诊断中，它还在GKD变体中减少了工具调用循环和重复调用。这些结果表明，多教师OPD应监控教师信号的作用位置，而不仅仅是其聚合大小。

## Abstract
Agentic language models must learn when to call tools, when to consume tool responses, and when to answer directly. This makes multi-teacher on-policy distillation a natural training strategy: one teacher can specialize in tool calls, another in direct responses, and the student can learn from both on   its own generated distribution. We show that this strategy can induce a behavior shift that is invisible from aggregate losses alone. In a two-teacher tool-use setting, vanilla generalized knowledge distillation improves tool-call recall but also moves the model toward over-calling, where it calls tools   on examples that should be answered directly. Aggregate explanations are insufficient: tool-call samples do not receive more token exposure, and full-sequence per-token divergence is not larger for the tool-call teacher. We instead analyze behavior leverage imbalance: local token-level signals at mode-   entry and structural positions, such as <tool_call> and function names, can have disproportionate control over the global generation mode. We propose Soft Clamp, a per-token divergence calibration method that dynamically compresses extreme token-level Jensen-Shannon divergence while preserving nonzero   gradients. On APIGen-MT, Soft Clamp reduces over-calling from 13.7% to 9.0% relative to vanilla GKD while matching its decision accuracy. In a BFCL multi-turn diagnostic, it also lowers tool-call loops and repeated calls among GKD variants. These results suggest that multi-teacher OPD should monitor   where teacher signals act, not only how large they are in aggregate.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在多教师在线策略蒸馏（Multi-Teacher On-Policy Distillation, MOPD）中，当两个教师分别专精于工具调用和直接回答时，学生模型可能出现“工具调用边界漂移”（Tool-Call Boundary Drift），即过度调用工具——在应直接回答的样本上也发起工具调用，而传统聚合损失（如平均 token 级散度）无法察觉此行为偏移。
- **研究动机**：智能体语言模型需要灵活决定何时调用工具、何时直接回答。多教师蒸馏被认为是一种自然策略（一个教师专精工具，另一个专精回答），但实际训练中教师信号的“行为杠杆不平衡”（behavior leverage imbalance）导致局部 token 级信号（如 `<tool_call>`、函数名等模式入口 token）对全局生成模式具有不成比例的控制力，从而引发边界漂移。现有聚合监控指标掩盖了该问题。
- **整体含义**：这项工作提出应审计教师信号的作用位置而非仅聚合大小，并引入一种新的训练时校准方法（Soft Clamp）来缓解漂移，同时暴露了不同校准策略之间的权衡。

## 2. 论文提出的方法论

- **核心思想**：识别出多教师 OPD 中工具/响应边界漂移源于极端 token 级散度的集中（少量 token 承载绝大部分散度），这些 token 往往位于模式入口或结构位置，具有高行为杠杆。因此，校准应直接压缩极端 token 级散度，同时保留梯度，而非全局重加权或硬截断。
- **关键技术细节**：
  - **Soft Clamp**：一种批自适应、局部的逐 token 散度压缩方法。
    - 对每个批次，计算动态阈值 \( C = \text{stopgrad}(k \cdot \text{mean}_i(d_i)) \)，其中 \( k \) 是钳位因子（默认 3.0），`stopgrad` 阻止阈值梯度反向传播。
    - 对每个 token 的 JSD 散度 \( d_i \)：
      - 若 \( d_i \le C \)，则 \( d'_i = d_i \)（保持不变）；
      - 若 \( d_i > C \)，则 \( d'_i = d_i \cdot \frac{C}{\text{stopgrad}(d_i)} \)（前向值被上限为 \( C \)，梯度按 \( C/d_i \) 缩放，保留非零梯度）。
    - 最终蒸馏损失为校准后 \( d'_i \) 的均值，与 SFT 锚点损失（超参数 \( \alpha=0.3 \)，仅用于数据集来源的轨迹）相加。
  - **算法流程**：
    1. 从学生当前策略采样轨迹（以概率 \( \lambda=0.8 \) 使用学生生成，否则使用数据集轨迹）。
    2. 对每个 token，获取教师分布（工具教师或回答教师，按教师标签路由）并计算 top-k 近似 JSD（使用教师返回的前 32 个 token 重归一化）。
    3. 应用 Soft Clamp：计算批级动态阈值 \( C \)，按上述规则调整每个 token 的散度。
    4. 计算校准后的蒸馏损失，并与 SFT 锚点损失结合（仅当轨迹来自数据集时）。
    5. 反向传播更新学生参数。
- **对比的其他校准策略**：
  - **Hard Clip**：固定阈值截断（\( c=0.5 \)），去除超出阈值 token 的梯度。
  - **Global Reweight**：批级相对重加权，使用 z-score 归一化后指数衰减权重，调整所有 token 的权重。
  - **Entry Bias**（推理时）：对第一个 `<tool_call>` token 添加负偏置（在验证集上选取），仅影响推理阶段。

## 6. 论文的主要结论与发现

- **核心发现**：多教师 OPD 会导致工具调用边界漂移——Vanilla GKD 提高调用召回率但显著增加过度调用（从 SFT 的 4.9% 升至 14.2%），而聚合损失（token 暴露量、平均 JSD）无法解释此偏移。
- **诊断结论**：漂移源于极少量 token（前 1% 占 41.2% 散度）在模式入口和结构位置的高行为杠杆；响应侧决策压力（`P(tool_call)`）与最终过度调用高度相关。
- **校准对比结论**：
  - **Soft Clamp** 在 9B 实验中实现最低过度调用（9.0%±0.2%），且观察方差最低，同时保持相近决策准确率（88.7%）；在 BFCL 多轮中减少调用循环（Loop@3 从 15.1% 降至 10.3%）、重复调用（17.7% 降至 11.3%），提升非工具最终率（87.3%→93.9%）。
  - **Global Reweight** 是强竞争者，但 Soft Clamp 在 APIGen-MT 过度调用和方差上更优；在 4B 复制中两者接近，Global Reweight 在多轮指标上略优。
  - **Hard Clip** 和 **Entry Bias** 也能部分缓解，但各有劣势（硬截断丢失梯度，入口偏置需验证集调参且无法完全复现联合行为）。
- **下结论**：多教师 OPD 应作为行为边界校准问题处理；监控教师信号的位置比聚合大小更重要。Soft Clamp 提供一种轻量、局部、保留梯度的训练时选项。

## 7. 优点

- **诊断视角创新**：提出“行为杠杆不平衡”概念，将边界漂移归因于局部高散度 token 的集中，而非简单的不平衡样本量或总散度，为理解和干预提供新方向。
- **方法轻量且有效**：Soft Clamp 仅需动态批阈值计算，开销极小；无需额外教师、奖励模型或验证集（相比推理时偏置），适应训练过程中散度尺度变化；保留极端 token 梯度，避免硬截断的“信息丢失”。
- **实验设计细致**：
  - 多组诊断验证（聚合统计失效、散度集中、检查点轨迹、响应侧决策压力）互相佐证机制。
  - 消融实验（Entry-k、Random-k、因子敏感度）排除简单入口 token 解释，支持更广泛的局部机制。
  - 推理时反事实进一步隔离边界效应，区分训练时与推理时校准。
  - 包含小尺寸复制增强泛化性。
- **结果报告清晰**：多次使用三随机种子+标准偏差，提供均值和方差，便于比较稳定性和统计意义。
- **代码与数据开源承诺**：计划发布训练代码、分析脚本、评估工具，促进可重复性。

## 8. 不足与局限

- **模型家族局限**：所有实验仅使用 Qwen3.5（9B 和 4B），未验证其他 LLM 架构（如 LLaMA、Mistral、Gemma），跨模型泛化性未知。
- **机制证据为诊断性**：作者明确承认 token 级散度集中、决策压力等为诊断证据，并非完整因果证明。Soft Clamp 的优势可能与局部令牌模式强相关，但尚未严格证明行为杠杆不平衡是唯一原因。
- **教师 API 截断近似**：使用的 JSD 仅基于教师返回的 top-32 token 重归一化，并非全词汇表散度，可能低估或高估部分 token 信号。
- **依赖 SFT 锚点**：主要 GKD 变体均使用 SFT 锚点（`sft_alpha=0.3`）以稳定格式。纯 OPD 实验（无锚点）导致格式崩塌，无法进行同条件对比，因此方法的真正抗漂移能力在无锚点场景下未充分验证。
- **BFCL 多轮诊断为自定义**：非官方 BFCL 排行榜协议；结果反映的是特定模拟环境下的行为，可能无法直接对应实际部署场景。
- **范围限制**：仅研究工具调用 vs 直接回答的二元边界。多教师 OPD 可能在其他技能集成（如推理 vs 记忆）中也有类似漂移，未探索。
- **校准策略对比深度**：Global Reweight 超参数（α, z_max, w_min/max）和 Hard Clip 阈值均基于单次诊断选择，未对每个种子单独调参；Soft Clamp 因子同样来自小规模扫描，可能不适用于不同数据或模型。
- **未见对其他教师路由策略的影响**：教师标签固定（按最终目标类型），未考虑路由策略本身可能引入的偏差。
