## 摘要
生成流网络（GFlowNets）为大型推理模型提供了一种有希望的替代最大化奖励的强化学习（RL）的方法，通过匹配奖励分布而不是坍缩到主导模式来鼓励多样化的推理路径。最近的工作在数学和代码方面显示出前景，但将GFlowNet风格的RL扩展到现代后训练流程仍然困难：随着模型规模、推演长度、奖励噪声和分布式系统复杂性的同时增长，学习到的提示条件划分函数成为梯度不稳定和工程开销的来源，而不是有用的归一化器。通过系统分析，我们发现先前被视为必需的学习到的划分函数，可以被从训练所需的推演组中计算出的批内蒙特卡洛估计所替代。我们提出了GFlowRL，一种简化的GFlowNet风格的RL算法，它完全移除辅助划分网络，同时保留奖励分布匹配目标，并通过两种稳定器完成：推演/训练器漂移的重要性采样校正和异常残差的不对称流间隙裁剪。GFlowRL在数学、代码和对抗性红队基准测试上超越了所有对比方法，在14B规模上达到了2048的Codeforces评分（在o3-mini的25 Elo以内），并在AdvBench和HarmBench上获得了最高的平均ASR@1，超越了之前最先进的多轮攻击者，而之前的GFlowNet风格方法FlowRL则发散。相同的方案也适用于所有评估的MoE配置，最高达235B参数，在这些配置中FlowRL同样无法收敛。据我们所知，GFlowRL是第一个在密集和稀疏架构上都能稳定扩展的GFlowNet风格RL算法。代码将在：https://github.com/microsoft/gflowrl

## Abstract
Generative Flow Networks (GFlowNets) offer a promising alternative to reward-maximizing reinforcement learning (RL) for large reasoning models, encouraging diverse reasoning paths by matching reward distributions rather than collapsing to dominant modes. Recent work shows promise on math and code, but scaling GFlowNet-style RL to modern post-training pipelines remains difficult: as model size, rollout horizon, reward noise, and distributed-systems complexity grow together, a learned prompt-conditional partition function becomes a source of gradient instability and engineering overhead rather than a useful normalizer. Through systematic analysis, we find that the learned partition function, previously treated as essential, can be replaced by an in-batch Monte Carlo estimate computed from the rollout group already required for training. We propose GFlowRL, a streamlined GFlowNet-style RL algorithm that removes the auxiliary partition network entirely while preserving the reward-distribution-matching objective, completed by two stabilizers: importance-sampling correction for rollout/trainer drift and asymmetric flow-gap clipping for outlier residuals. GFlowRL exceeds all counterparts on math, code, and adversarial red-teaming benchmarks, reaching a Codeforces rating of 2048 at the 14B scale (within 25 Elo of o3-mini) and attaining the highest average ASR@1 on AdvBench and HarmBench, outperforming the previous SOTA multi-turn attacker in a regime where FlowRL, a prior GFlowNet-style method, diverges. The same recipe transfers to all evaluated MoE configurations up to 235B parameters, where FlowRL again fails to converge. To our knowledge, GFlowRL is the first GFlowNet-style RL algorithm to scale stably across both dense and sparse architectures. Code will be at: https://github.com/microsoft/gflowrl

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前大型语言模型（LLM）后训练中，主流强化学习（RL）方法如PPO、GRPO等通过最大化期望奖励来优化策略，这会导致模式坍缩（mode collapse），即模型倾向于集中概率质量在少数高奖励的推理路径上，丧失推理多样性。生成流网络（GFlowNets）通过匹配奖励分布（即按奖励比例采样）来鼓励多样化推理路径，理论上能解决此问题。但将GFlowNet风格的RL扩展到现代LLM后训练时遇到重大困难：模型规模、推理链长度、奖励噪声和分布式系统复杂性同步增长，导致学习到的提示条件划分函数（partition function）成为梯度不稳定和工程开销的根源，而非有用的归一化器。

- **核心问题**：先前的GFlowNet风格方法（如FlowRL、FOR）依赖一个辅助的神经网络来学习划分函数，但该网络与预训练的策略模型之间存在严重的“学习周期不匹配”：策略模型有数十亿参数且只需要微调，而划分网络是随机初始化的，需要在同样有限的训练步内从零学习复杂的目标，导致梯度爆炸、训练不稳定，且实际上并未提供有用信号。

- **整体含义**：论文挑战了GFlowNet文献中“划分函数必须被学习”的共识，提出可以通过批内蒙特卡洛估计来替代学习到的划分函数，从而在保留分布匹配目标的同时大幅提升稳定性。这是首次实现GFlowNet风格RL在密集和稀疏（MoE）架构上均稳定扩展的工作。

## 2. 论文提出的方法论

### 核心思想
用批内蒙特卡洛估计替代学习到的分区网络，并引入两个稳定机制：重要性采样校正和异步流间隙裁剪。

### 关键技术细节

1. **移除学习到的分区网络**：彻底去掉FlowRL等中使用的3层MLP分区网络（参数化为Zφ(x)）。

2. **批内蒙特卡洛估计**：在GRPO风格的训练中，每个提示已经采样了G个推演（rollout）。利用这些推演来估计对数分区函数：
   ```
   Ẑ_t(x) = (1/G) * Σ_i [β·r(x, y_i) + log π_ref(y_i | x) - log π_old(y_i | x)]
   ```
   其中β是逆温度超参数，r是奖励，π_ref是冻结的参考策略，π_old是推演策略（模型快照）。这个估计直接使用已有推演，不引入新参数。

3. **分布匹配残差**：对于每个推演y_i，定义残差为：
   ```
   Δ(i)(θ) = sg[Ẑ_t(x)] + (1/|y_i|) * log[π_θ(y_i|x)/π_ref(y_i|x)] - β·r(x, y_i)
   ```
   sg[·]表示停止梯度，Ẑ_t(x)仅作为基线用于居中残差，本身不携带梯度。长度归一化防止长序列主导损失。

4. **不对称流间隙裁剪**：将残差中的“流间隙”（即基于推演策略的分布匹配误差）进行裁剪：
   ```
   g̃(i) = clip(g(i), -ε_low, +ε_high)
   ```
   其中ε_low < ε_high，允许正校正比负校正更大。这在数学推理中特别有用：早期正确解决方案可能被欠采样，更新应更积极地向上推概率质量。

5. **重要性采样校正**：权重w(i) = min(π_θ(y_i|x)/π_old(y_i|x), 1+ε) 用于校正推演策略与当前策略之间的偏移。

6. **最终的GFlowRL训练损失**（规范化版本）：
   ```
   L_GFlowRL = (1/G) * Σ_i w(i) * [g̃(i) + (1/|y_i|) * log(π_θ(y_i|x)/π_old(y_i|x))]²
   ```

### 理论保证
- 当裁剪不激活时，目标保留了与轨迹平衡（Trajectory Balance）相同的固定点：最优策略满足π_θ(y|x) ∝ π_ref(y|x)·exp(β·r(x,y))，即奖励比例采样。
- 在最优解附近裁剪自然不激活，不扰动平稳分布。
- 长度归一化引入轻微近似，但在推演长度大致均匀时影响可忽略。
