## 摘要
智能体强化学习可能导致工具滥用，即模型过度使用外部工具，即使对于内部推理可解决的查询也是如此。现有方法通过统一的工具使用惩罚或硬限制来缓解此问题，这降低了工具使用频率，但也可能抑制有用的工具辅助探索。我们提出EAPO，一种高效智能体策略优化框架，学习选择性工具使用。EAPO在每个滚动组中引入无工具轨迹，应用难度感知的奖励塑造主要对较简单查询惩罚冗余工具调用，并使用置信度感知的令牌重新加权改进策略学习。在九个数学和知识密集型推理基准上，EAPO在Qwen2.5-3B、Qwen2.5-7B和Llama3.1-8B上一致改善了准确率-效率权衡。与GRPO相比，EAPO平均性能提升10.45%、7.27%和9.69%，同时平均工具调用分别减少18.33%、18.33%和24.59%。这些结果表明，智能体可以学习何时不使用工具，而不损害工具集成推理。

## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在基于强化学习（RL）训练智能体使用外部工具（如搜索引擎、代码解释器）时，模型容易产生 **“工具滥用”（Tool Abuse）** 行为。具体表现为：模型倾向于过度依赖外部工具，即使对于非常简单的、仅凭自身推理即可完成的任务，也会习惯性地调用工具。
- **研究动机**：
    - **效率低下**：不必要的工具调用增加了计算开销、延迟和部署成本（尤其对于按次计费的API）。
    - **现有方法的局限性**：现有缓解方法（如统一施加工具使用惩罚或设定硬限制）虽然降低了工具使用频率，但无差别地惩罚所有工具调用，会导致模型在真正需要工具辅助的困难任务上也抑制探索，反而损害了工具增强推理的初衷。
    - **关键矛盾**：核心问题不在于“如何让模型更少使用工具”，而在于“如何让模型**学会区分何时该用、何时不该用**工具”。

#### 2. 论文提出的方法论：EAPO 框架

- **核心思想**：提出一种名为**高效智能体策略优化（Efficient Agentic Policy Optimization, EAPO）** 的框架，旨在让智能体学会**选择性**使用工具，即在简单任务上依赖内部推理，在困难任务上则利用工具辅助。
- **关键技术细节**：
    1.  **效率感知滚动（Efficiency-Aware Rollout）**：
        - 在训练阶段的每个批次中，强制生成一定数量（至少 `n` 条）的**无工具轨迹**。即通过修改提示词和禁用工具执行，让模型尝试仅靠自身推理完成任务。
        - 这些无工具轨迹为模型提供了直接证据，证明该查询是否可以被内部推理解决，从而帮助区分“必要工具使用”和“工具滥用”。
        - 定义一个 **效率系数** `ε(T_i)` 衡量轨迹 `T_i` 的工具使用冗余度：`ε(T_i) = exp(-β(c_i - c_min))`。其中 `c_i` 是轨迹中的工具调用次数，`c_min` 是所有正确轨迹中最少的工具调用次数，`β` 是惩罚强度超参数。该系数会作为奖励的调节因子。

    2.  **难度感知奖励塑造（Difficulty-Aware Reward Shaping）**：
        - 基于查询的难度动态调整奖励。查询难度 `d(q)` 定义为：`d(q) = 1 - (正确轨迹占比)`。一个查询被越多的轨迹正确回答，其难度越低。
        - 最终的奖励函数 `R̃_i` 如下：
          `R̃_i = [ d(q) + (1 - d(q)) * ε(T_i) ] * R_i`
          - 对于**简单查询**（`d(q)` 小）：效率系数 `ε(T_i)` 的权重更大，即那些使用了过多工具的轨迹会受到更严厉的惩罚。
          - 对于**困难查询**（`d(q)` 大）：原始奖励 `R_i` 的权重更高，鼓励模型去探索和使用工具，而不因工具调用过多而被过度惩罚。这使得优化过程对工具惩罚不敏感，避免了抑制困难任务上的探索。

    3.  **置信度感知令牌重加权（Confidence-Aware Advantage Reweighting）**：
        - 基于生成令牌的置信度（即其对数概率 `log p_t(y_t)`）来调整策略梯度。高置信度（值接近0）的令牌信息密度低，低置信度（负值大）的令牌信息密度高。
        - **策略**：当轨迹正确时，**提升**那些低置信度（信息量大）令牌的学习权重；当轨迹错误时，**降低**那些高置信度（错误结论）令牌的权重。
        - 这通过一个令牌级别的权重 `ω_i,u,t` 实现，最终重新加权的优势函数为 `A_i,u,t = ω_i,u,t * A_i`，其中 `A_i` 是轨迹级别的优势。

- **算法流程**（文字说明）：
    1.  从数据集中采样一个批次。
    2.  对每个查询，先强制生成 `n` 条无工具轨迹，再生成 `M-n` 条允许使用工具的轨迹，共同组成 `M` 条轨迹的补充组。
    3.  计算每条轨迹的初始奖励 `R_i`（基于答案正确性和格式）。
    4.  计算查询难度 `d(q)` 和每条轨迹的效率系数 `ε(T_i)`。
    5.  根据公式 6 计算调整后的奖励 `R̃_i`。
    6.  基于调整后的奖励计算每条轨迹的优势值 `A_i`。
    7.  计算每个参与学习令牌的置信度，并依据公式 8 为其分配权重 `ω_i,u,t`。
    8.  使用重新加权后的优势值 `A_i,u,t` 和 KL散度正则化项更新策略模型。
