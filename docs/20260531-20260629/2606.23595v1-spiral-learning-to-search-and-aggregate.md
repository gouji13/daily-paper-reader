## 摘要
语言模型推理可以通过在测试时使用脚手架来显著提高，该脚手架在不同原语之间扩展推理计算——轨迹内的顺序推理、独立采样的并行轨迹，以及将多个推理轨迹聚合为最终响应。然而，在后训练阶段，语言模型仅针对单个轨迹内的顺序推理进行优化。我们引入了顺序-并行-聚合强化学习（SPIRAL），这是一个框架，其中语言模型被训练使用所有三种原语，作为统一推理计算流水线的一部分。具体来说，语言模型首先并行采样一组独立轨迹，每个轨迹通过顺序思维链推理产生，然后基于这些轨迹生成最终的聚合轨迹；所有组件根据最终聚合响应的奖励进行端到端优化。为了训练该系统，SPIRAL使用集合强化学习来教模型产生一组对聚合器集体有用的轨迹，并使用标准强化学习来教模型将集合聚合为改进的最终响应。我们在推理任务上的实验表明，SPIRAL有效地随推理计算扩展，在扩展所有三种计算原语时，其扩展效率比GRPO高出高达11倍，性能高出15%。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：语言模型在测试时可以通过扩展推理计算提升性能，但现有方法主要依赖手工设计的脚手架（scaffolds），分为三种原语：单个轨迹内的顺序推理、独立采样的并行轨迹、以及将多条轨迹聚合为最终答案。当前后训练（post-training）仅优化顺序推理（如单条思维链），导致模型在测试时无法有效利用并行采样和聚合这两个原语。
- **核心问题**：如何训练语言模型学会自主协调并最大化利用顺序、并行、聚合三种推理计算原语，从而弥合训练与测试时的差距，避免手工设计复杂脚手架。
- **整体含义**：提出 SPIRAL 框架，通过端到端强化学习统一优化所有原语，使模型能够在测试时更高效地扩展计算资源，在推理任务上显著提升性能。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将推理计算分为三个原语：顺序计算（单轨迹内思考）、并行计算（独立采样多条轨迹）、聚合计算（基于多条轨迹生成最终答案）。在训练时让模型同时暴露于这三种原语，并使用联合奖励（最终答案的正确性）进行优化，从而学习如何生成对聚合有用的并行轨迹，以及如何有效聚合它们。
- **关键技术细节**：
    - **问题形式化**：优化目标为 $\max_{\theta,\phi} \mathbb{E}_{y_{1:n}\sim\pi_\theta(\cdot|x)}\mathbb{E}_{y^*\sim\pi_\phi(\cdot|x,y_{1:n})}[r(x,y^*)]$，其中 $y_{1:n}$ 是并行搜索轨迹，$y^*$ 是聚合轨迹，$r$ 是最终答案奖励。
    - **算法流程**（同一模型 $\theta=\phi$）：
        1. 对每个输入 $x$，采样 $N_1$ 条搜索轨迹 $y_1,\dots,y_{N_1} \sim \pi_\theta(\cdot|x)$。
        2. 从 $\binom{N_1}{n}$ 个无序集合中均匀采样 $K$ 个大小为 $n$ 的集合 $G_1,\dots,G_K$。
        3. 对每个集合 $G_i$，采样 $N_2$ 条聚合轨迹 $y^{G_i}_1,\dots,y^{G_i}_{N_2} \sim \pi_\theta(\cdot|x,G_i)$。
        4. 计算每个集合的得分 $\hat{f}_{\text{spiral}}(x,G_i)=\frac{1}{N_2}\sum_{j} r(x,y^{G_i}_j)$，并计算集合基线 $\hat{f}_{\text{spiral}}(x)=\frac{1}{K}\sum_i \hat{f}_{\text{spiral}}(x,G_i)$。
        5. 集合优势 $A^\sharp(x,G_i)=\hat{f}_{\text{spiral}}(x,G_i)-\hat{f}_{\text{spiral}}(x)$；搜索轨迹的边际集合优势 $A^\sharp_{\text{marg}}(x,y_j)=\frac{1}{|\mathcal{G}(y_j)|}\sum_{G\ni y_j} A^\sharp(x,G)$。
        6. 聚合轨迹的优势采用标准 RL 的组内基线：$A(x,y^{G_i}_j)=r(x,y^{G_i}_j)-\frac{1}{N_2}\sum_k r(x,y^{G_i}_k)$。
        7. 采用策略梯度更新，搜索轨迹部分使用集合 RL 梯度，聚合轨迹部分使用标准 RL 梯度。
    - **关键创新**：将并行轨迹的生成视为集合优化问题，通过集合 RL 的共享学习信号鼓励轨迹多样性且对聚合有用；聚合阶段通过标准 RL 学习如何有效合成信息。

### 3. 实验设计：使用的数据集、Benchmark、对比方法

- **数据集**：使用 POLARIS-53k 数学推理数据集的过滤子集（论文未明确说明具体过滤方式）。训练时每个 batch 256 个问题，每个问题 24 个 rollouts，共训练 2 个 epoch。
- **基准模型**：Qwen3-4b-Instruct-2507。
- **对比方法**：主要对比 GRPO（Group Relative Policy Optimization），也对比了多数投票（majority voting）和递归自聚合（recursive self-aggregation）。在训练计算量上保证公平（每问题 token 预算相同，SPIRAL 每问题 98304 tokens，GRPO 也 98304 tokens）。
