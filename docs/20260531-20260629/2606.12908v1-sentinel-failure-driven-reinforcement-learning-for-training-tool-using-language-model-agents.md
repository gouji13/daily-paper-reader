## 摘要
语言模型智能体通过多轮工具使用在解决现实任务方面越来越有效。然而，训练可靠的工具使用智能体在实践中仍然具有挑战性。虽然强化学习提供了一种从自身环境交互中改进智能体的在策略范式，但其有效性在很大程度上取决于训练任务分布。当任务在训练前固定时，任务分布会与策略不断变化的能力越来越不匹配，导致许多轨迹浪费在无信息的任务上。我们提出SENTINEL，一种失败驱动的强化学习框架，将求解器的轨迹失败转化为针对性的训练任务。SENTINEL遵循控制器-提议器-求解器循环：控制器分析失败轨迹并总结重复出现的错误模式，提议器生成强调这些弱点的可执行任务，求解器在针对性任务上进行训练。在Tau2-Bench Retail与Qwen3-4B-Thinking-2507上，SENTINEL将Pass^1从66.4提升到74.9，并且在Pass^k指标上的通用合成任务中优于RL。这些结果表明，模型失败为改进工具使用语言模型智能体提供了有效且可扩展的针对性训练信号来源。

## Abstract
Language model agents are increasingly effective in solving realistic tasks through multi-turn tool use. However, training reliable tool-using agents remains challenging in practice. While reinforcement learning provides an on-policy paradigm for improving agents from their own environment interactions, its effectiveness depends heavily on the training task distribution. When tasks are fixed before training, the task distribution can become increasingly mismatched with the policy's evolving capabilities, causing many rollouts to be spent on uninformative tasks. We propose SENTINEL, a failure-driven reinforcement learning framework that turns the Solver's rollout failures into targeted training tasks. SENTINEL follows a Controller--Proposer--Solver loop: the Controller analyzes failed trajectories and summarizes recurring error patterns, the Proposer generates executable tasks that stress these weaknesses, and the Solver is trained on the targeted tasks. On Tau2-Bench Retail with Qwen3-4B-Thinking-2507, SENTINEL improves Pass\^{}1 from 66.4 to 74.9 and outperforms RL on general synthetic tasks across Pass\^{}k metrics. These results demonstrate that model failures provide an effective and scalable source of targeted training signal for improving tool-using language model agents.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：训练工具使用语言模型智能体时，强化学习（RL）的有效性高度依赖于训练任务分布。当任务在训练前固定不变时，任务分布会与策略不断变化的能力产生错配，导致大量 rollout 被浪费在无信息任务（过于简单、过难或重复）上，从而降低学习效率。
- **研究动机**：现有 RL 方法大多依赖静态任务集，无法自适应地暴露和解决当前策略的弱点。作者提出：失败轨迹是连接诊断模型弱点与生成训练数据的天然桥梁——模型自身的失败揭示了它尚未学会的能力。
- **整体含义**：提出一种**失败驱动的强化学习框架 SENTINEL**，通过分析当前策略的失败轨迹来指导下一轮训练任务的生成，使训练分布随模型能力动态调整，提升工具使用智能体的可靠性。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：构建“控制器（Controller）—提议器（Proposer）—求解器（Solver）”闭环循环。求解器在工具环境中执行 rollout；控制器分析失败轨迹并总结错误模式；提议器根据错误模式生成针对性训练任务；求解器在新任务上通过 RL 优化。更新后的求解器再次 rollout，形成迭代。
- **关键技术细节**：
  - **控制器**：每隔固定 rollout 批次触发。对轨迹去重后分离失败与成功轨迹，用 LLM 分析失败轨迹，输出 K 个错误模式（如错误物品选择、缺失状态变更动作、策略违规等）。同时维护训练历史，区分持续错误与已解决错误，生成自然语言指令传递给提议器。
  - **提议器**：基于轨迹接地（trajectory-grounded）方式生成任务。先在环境中构建可执行的参考轨迹（确保状态可达），再根据控制器指令将轨迹重写为用户请求，使任务专门针对当前失败模式（例如对物品选择失败，生成含相似变体的场景）。
  - **求解器**：使用 GRPO（Group Relative Policy Optimization）进行 RL 训练。奖励包括最终任务成功信号（数据库状态检查和通信检查）以及多个塑形项（动作级验证、重复工具调用惩罚、格式惩罚、长度惩罚）。训练时维护一个任务缓冲区，包含原始任务和提议器生成的新任务。
- **算法流程**（文字描述）：
  1. 求解器在工具环境 E 中执行多轮 rollout，收集轨迹。
  2. 控制器抽取失败轨迹，进行去重、失败模式分析，生成生成指令。
  3. 提议器在环境中构建可行参考轨迹，参考指令将其重写为针对性用户任务。
  4. 将新任务注入训练缓冲区，求解器使用 GRPO 更新策略。
  5. 重复步骤 1-4。

## 6. 主要结论与发现

- **主要结论**：失败驱动的强化学习（SENTINEL）显著优于使用固定通用任务的强化学习。在 Pass^1 上，SENTINEL 从基线 66.4 提升至 74.9，而通用 RL 仅提升至 69.0。所有 Pass^k 指标上 SENTINEL 均最优。
- **重要发现**：当模型已经通过 SFT 达到较高性能（74.3）时，直接应用通用 RL 反而导致性能下降（降至 68.1），但 SENTINEL 能进一步将其提升至 78.1，表明针对性任务生成比大量通用任务更有效，且可避免模型退化（如工具循环、奖励捷径利用）。
- **失败模式分析**：控制器发现的错误集中于精确工具使用（如物品接地、多记录推理、缺失动作、执行控制），而非简单工具选择错误，且错误类型随训练动态变化（从直接动作缺失转向策略级错误）。

## 7. 优点：方法或实验设计的亮点

- **方法亮点**：
  - **闭环自适应**：将失败诊断、任务生成与 RL 优化紧密耦合，使训练分布随模型弱点动态调整。
  - **轨迹接地任务生成**：先构建可执行参考轨迹再写用户请求，确保任务可行性，避免答案不一致。
  - **奖励塑形**：加入动作验证和多种惩罚，减少奖励捷径和退化行为。
  - **控制历史记忆**：控制器维护训练历史，区分持久错误与已解决错误，避免重复生成。
- **实验设计亮点**：
  - 在 SFT 初始化模型上验证，展示了即使模型已较强，针对性任务仍能带来提升，而通用任务有害。
  - 可视化失败模式类型及其动态变化，提供了方法有效性的定性支持。

## 8. 不足与局限

- **实验覆盖不足**：仅在 Tau2-Bench Retail 一个域上测试，未在其他工具域（如软件工程、Web 代理、MCP 环境）验证泛化性。
- **生成任务覆盖率有限**：任务生成依赖于当前求解器暴露的失败，可能遗漏罕见或未采样场景的失败模式，导致对某些测试用例覆盖不全。
- **偏差风险**：控制器和分析器均使用 LLM，存在生成模式偏差或错误模式误判风险；提议器也可能过度生成特定类型任务，导致分布偏移。
- **计算开销**：闭环循环需要多次 rollout、分析和任务生成，实际训练可能比静态 RL 更耗时。
- **未报告方差**：主要结果未提供多次运行的标准差或置信区间，难以评估稳定性。
- **未进行超参数消融**：如触发频率、最大任务数、奖励阈值等参数的影响未探讨。
