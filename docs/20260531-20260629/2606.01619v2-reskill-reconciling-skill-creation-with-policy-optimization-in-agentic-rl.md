## 摘要
自主强化学习使LLM智能体能够从环境奖励中持续改进，但由此产生的策略并未系统地积累可跨任务泛化的可复用策略。模块化技能可以提供此类可复用策略，然而现有的技能增强强化学习方法将技能创建与策略优化解耦，存在采用与演进策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出了ReSkill，一个将强化学习融入循环的技能创建框架，协调技能演化与策略学习。ReSkill利用GRPO的分组结构，以极小的额外开销自然嵌入三种机制：（1）基于断言的技能创建器，从过去经验中诊断失败并提出有条件的、基于触发的技能修订；（2）组内 rollout 采样，实现技能版本的可控比较，捕捉哪个版本最支持策略的持续学习；（3）采用自适应折扣的汤普森采样，在策略演进中平衡技能版本选择的探索与利用。在多个领域中，ReSkill始终优于现有的基于记忆和技能的强化学习方法，在未见过的任务上提升最大。技能生命周期分析表明，随着策略改进，技能会自动创建、测试、优化和修剪，展示了协调的技能-策略共同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：利用 GRPO 训练中每个组内 rollouts 共享同一任务和策略的特点，为组内不同 rollout 分配不同技能版本（旧版 `S_old` 或新版 `S_new`），从而在不增加额外 rollout 规模的情况下同时实现策略优化、失败诊断和技能评估。三个关键技术机制：
  - **断言驱动的技能创建器（Assertion-Driven Skill Creator）**：从经验库中诊断失败模式，使用动态断言集（rule-based predicates）对轨迹进行批量化评分，再由 LLM 分析器生成排名后的失败诊断，并据此提出条件式、基于触发（trigger）的技能修订（ADD/MODIFY/DELETE）。
  - **组内 rollout 采样（Within-Group Rollout Sampling）**：在 GRPO 的每个组内，通过 Thompson Sampling 为每个 rollout 分配技能版本，从而在同一组内实现版本间的可控比较，捕捉哪个版本更有利于当前策略的学习。
  - **自适应折扣的 Thompson 采样（Adaptive Thompson Sampling）**：将新旧技能版本视为两个臂，使用 Beta 分布建模其成功概率。在每步后，根据该步的样本量 `nt` 对历史伪计数进行折扣（折扣因子 `wt = 1/(1 + nt/M)`），然后更新后验。在每步用蒙特卡洛估计 `P(p_new > p_old)` 并裁剪得到分配概率 `π_t(new)`，最后独立采样分配。在每个演化周期结束时，根据折扣后验的期望值接受或拒绝新版本。
- **关键技术细节**：

  - **断言集动态更新**：由 LLM 分析器根据观察到的通过率添加、修改或删除断言，使其与策略的失败模式变化对齐。

## 6. 主要结论与发现
- **一致超越**：ReSkill 在所有基准上均优于现有方法，包括 GRPO、MemRL、Evolver、INSPO、SkillRL 等。例如 ALFWorld 4B 总体 89.8% vs 最强基线 SkillRL 83.9%；搜索 4B 47.6% vs SkillRL 45.1%。

- **技能与策略自动协同进化**：训练过程中观察到技能自动创建、测试、精炼和剪枝（如图 7），早期建立基础技能，后期修剪已内化的技能。
- **消融验证机制重要性**：无测试的自动接受导致性能崩溃（ALFWorld 62.7% vs ReSkill 89.8%），说明测试关键；协进化顺序（先技能后策略或先策略后技能）效果差；自适应折扣优于最后一步集或均匀加权。
- **测试时跨域适应**：冻结策略后，ReSkill 能通过仅更新技能库在 ScienceWorld 上快速适应（图 5），而基线几乎为零，说明协调训练培养了通用技能遵循能力。

- **机制设计精巧**：

- **实验全面**：覆盖具身推理、搜索、科学、编码、购物五个领域，对比多类基线（非RL、纯技能、RL变体，共超过 10 种方法），并在两个模型规模和多个 unseen 难度上进行验证。

## 8. 不足与局限
- **进化频率敏感**：在 ALFWorld 上，当演化频率过低（如 Ke=20）时性能下降（图 9），需要针对不同训练长度调整。论文建议自适应演化调度作为未来方向。
- **技能调用方式**：当前基于触发条件的自动加载给模型较少控制权，可能导致模型将技能名称误作环境动作（附录 F 案例“meta-bug”）或过度应用。未来可探索工具调用（tool-call）形式。
- **认知负荷**：小模型（4B）容易混淆技能指导与动作指令，或在不适用的情境下持续使用技能，反映出技能-策略融合需要更精细的奖励信号训练。
- **计算资源细节不完整**：虽给出了 GPU 数量和训练步数，但未提供准确的总训练时长或 GPU 时数，影响算力可复现性评估。
- **模型泛化性**：仅在 Qwen3-4B/8B 上评估，未在更大模型（如 70B）或其他族（如 LLaMA）上验证，结论的普遍性有待进一步确认。
- **技能创建器依赖外部 LLM**：尽管消融显示用基模型也能取得较好结果（ALFWorld 84.7% vs 89.8%），但仍有差距，且所有对比方法均使用相同的 Claude API 进行技能/记忆创建，可能对依赖该特定 LLM 能力的结论存在偏差风险。
