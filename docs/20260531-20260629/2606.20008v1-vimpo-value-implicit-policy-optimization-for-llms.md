## 摘要
基于可验证奖励的强化学习已成为提升大型语言模型推理能力的核心工具，但当前方法在简洁性与信用分配之间面临权衡。基于组相对的方法（如GRPO）避免了训练评论家，但通常为每个令牌分配轨迹级优势。行动者-评论家方法提供了更密集的学习信号，但需要学习价值函数，这本身存在训练不稳定性。我们提出VIMPO，一种免评论家的策略优化方法，它从KL正则化强化学习的最优性条件中推导出策略隐含的价值函数。对于自回归生成，所得价值递归可以用策略-参考对数比率表示，并由轨迹结束时无未来奖励的终端条件锚定。这给出了一个简单的价值损失，它整合了结果级别的可验证奖励，而无需训练评论家。同样的推导还产生了免评论家的行动者优势，使得VIMPO能够通过价值损失分离奖励整合，并通过类似PPO的行动者更新进行策略改进。在数学RLVR基准测试中，VIMPO在MATH-500、AIME 2024、AIME 2025和OlympiadBench上均优于GRPO，在竞赛式评估中尤其有更大增益。在噪声奖励下，VIMPO保持了对GRPO的一致优势，表明策略隐含价值优化可以在保持免评论家训练的实际简洁性的同时，提供更精细的信用分配。

## Abstract
Reinforcement learning with verifiable rewards has become a central tool for improving the reasoning ability of large language models, but current methods face a trade-off between simplicity and credit assignment. Group-relative methods such as GRPO avoid training a critic, but typically assign a trajectory-level advantage to every token. Actor-critic methods provide denser learning signals, but require a learned value function with its own training instability. We introduce VIMPO, a critic-free policy optimization method that derives a policy-implied value function from the optimality conditions of KL-regularized reinforcement learning. For autoregressive generation, the resulting value recurrence can be written in terms of policy-reference log-ratios and anchored by the terminal condition that no future reward remains at the end of a trajectory. This gives a simple value loss that incorporates outcome-level verifiable rewards without training a critic. The same derivation also yields a critic-free actor advantage, allowing VIMPO to separate reward incorporation through the value loss from policy improvement through a PPO-style actor update. On mathematical RLVR benchmarks, VIMPO improves over GRPO across MATH-500, AIME 2024, AIME 2025, and OlympiadBench, with especially larger gains on competition-style evaluations. Under noisy rewards, VIMPO retains a consistent advantage over GRPO, suggesting that policy-implied value optimization can provide finer credit assignment while preserving the practical simplicity of critic-free training.

  - **演员-评论家方法**（如 PPO、VAPO）提供密集的 token 级学习信号，但需要学习一个价值函数，这会引入额外的不稳定性（如评论家与策略的共适应、易崩溃）。

### 2. 方法论：核心思想、关键技术细节

- **核心思想**：将自回归生成建模为**确定性转移的 MDP**，从 KL 正则化 RL 的最优性条件出发，推导出**策略隐含的价值函数**，无需学习独立的评论家网络。该价值函数仅依赖于策略与参考模型的 log‑ratio 及终端边界条件（轨迹结束时价值为零），然后通过 Bellman 一致性损失进行优化。同一推导还给出了闭式的单步 TD 优势，可无缝集成到 PPO 风格的行动者更新中。

- **关键技术细节**：
  1. **确定性 MDP 与 KL 正则化优化**：对状态 $s_t$（已生成的前缀），最优策略 $\pi^*$ 满足：
     $$
     \pi^*(a|s_t) = \frac{\pi_{\text{ref}}(a|s_t)}{Z(s_t)} \exp\left(\frac{1}{\beta} Q^*(s_t, a)\right)
     $$
  2. **价值递归推导**：结合贝尔曼方程 $Q^*(s_t, a_t) = r(s_t, a_t) + \gamma V^*(s_{t+1})$，消去 $\ln Z$ 后得到核心恒等式：
     $$
     \beta \ln\frac{\pi^*(a_t|s_t)}{\pi_{\text{ref}}(a_t|s_t)} = r(s_t, a_t) + \gamma V^*(s_{t+1}) - V^*(s_t) + \beta \,\text{KL}^*(s_t)
     $$
     其中 $\text{KL}^*(s_t)=\text{KL}(\pi^*(\cdot|s_t)\|\pi_{\text{ref}}(\cdot|s_t))$。
  3. **策略隐含价值函数的构造**：将训练中的策略 $\pi_\theta$ 替代 $\pi^*$，并利用终端条件 $V(s_T)=0$，得到免评论家的价值损失：
     $$
     L_V(\pi) = \frac12 \left[ \sum_{k=0}^{T-1} \big(\beta\ln\frac{\pi(a_k|s_k)}{\pi_{\text{ref}}(a_k|s_k)} - \beta\,\text{sg}[\text{KL}^\pi(s_k)]\big) - (R_{\text{final}} - \bar R) \right]^2
     $$
     其中 $\text{sg}$ 为停止梯度，$\bar R$ 为组平均奖励。该损失将逐 token 的累积 log‑ratio 逼近归因后的结果奖励。
  4. **行动者优势**：由同一递推给出闭式单步 TD 优势 $\hat A_t^{\text{TD}} = \beta\ln\frac{\pi_\theta(a_t|s_t)}{\pi_{\text{ref}}(a_t|s_t)} - \beta\text{KL}^{\pi_\theta}(s_t)$，可进一步用 GAE 形式汇总为 $\hat A_t^\lambda$，然后用于 PPO 裁剪后的行动者损失 $L_A$。
  5. **总体目标**：$L_{\text{VIMPO}} = L_V + c_A L_A$。奖励信息只通过价值损失传入，行动者优势仅由策略内部量驱动，实现了奖励整合与策略改进的分离。

- **算法流程**（文字描述）：
  - 每步从策略采样一组 rollout（多个 prompt，每 prompt 多条轨迹）。
  - 计算每条轨迹的最终奖励与组平均。
  - 对每个 token 计算 log‑ratio、KL 项，进而得到策略隐含优势。
  - 利用终端条件构建价值损失 $L_V$。
  - 对优势进行标准化处理，计算 PPO 行动者损失 $L_A$。
  - 联合更新策略参数 $\theta$。

### 7. 优点

- **方法创新性**：将 KL 正则化最优性的恒等式巧妙地转化为免评论家的价值函数与优势，同时汇聚了两种范式的优点（简单训练 + 密集信号）。
- **奖励与策略改进的分离**：价值损失负责引入外部奖励，行动者优势仅由策略内部量（log‑ratio）驱动，这一设计在理论上可降低奖励噪声的影响。
- **训练稳定性**：无需训评论家网络，避免了评论家-策略共适应和值函数崩溃问题。
- **实验证据充分**：在多个标准基准上验证并给出详细动态曲线，噪声实验和消融实验进一步支撑了方法的有效性。
- **开源代码**：提供 GitHub 仓库，便于复现。
