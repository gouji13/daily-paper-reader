## 摘要
智能体强化学习使大语言模型智能体能够从环境奖励中持续改进，但由此产生的策略并未系统地积累可跨任务泛化的可复用策略。模块化技能可提供此类可复用策略，然而现有的技能增强强化学习方法将技能创建与策略优化分离，存在采用与进化策略相冲突的技能的风险。受Anthropic的Skill Creator启发，我们提出了ReSkill，一种强化学习在环的技能创建框架，可调和技能进化与策略学习。ReSkill利用GRPO的组级结构，以仅微小的额外开销自然嵌入三种机制：（1）基于断言的技能创建器，从过去经验中诊断失败并提出条件性的、基于触发的技能修订；（2）组内轨迹采样，实现技能版本的可控比较，捕捉哪个版本最能支持策略的持续学习；（3）采用自适应折扣的汤普森采样，在策略进化过程中平衡技能版本选择的探索与利用。在多个领域中，ReSkill始终优于现有的基于记忆和技能的强化学习方法，在未见任务上提升最大。对技能生命周期的分析显示，随着策略改进，技能会被自动创建、测试、完善和修剪，展示了调和的技能-策略共同进化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在大语言模型（LLM）智能体强化学习（Agentic RL）中，策略通过环境奖励不断优化，但难以系统性地积累可跨任务泛化的可重用策略（模块化技能）。现有技能增强的RL方法将技能创建与策略优化相分离，导致采纳的技能可能与不断进化的策略发生冲突，例如外部生成的技能未经过策略训练阶段的检验就被引入，可能损害学习效果。
- **研究背景**：
  - Agentic RL（如基于GRPO的训练）已展现出从奖励中学习的能力，但策略本身不自动积累模块化知识。
  - Anthropic的Skill Creator开创了通过迭代人类反馈自动创建模块化技能的方法，但其运行于静态策略和推理阶段，无法与策略训练动态融合。
  - 近期方法（如MEMRL、EvolveR、SkillRL）尝试在RL训练期间积累经验知识，但技能创建与策略优化仍作为独立过程进行，缺乏原则性测试，存在技能-策略不协调的风险。
- **研究含义**：提出一种**RL-in-the-loop的技能创建框架**，使技能创建与策略学习相互调和，每个训练步骤中的rollout同时服务于策略优化、失败诊断和技能版本评估，实现技能和策略的协同进化。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：利用GRPO训练中固有的**组级（group-wise）结构**，将技能版本分配在每个组内进行，使得同一任务、同一策略下不同rollout仅因技能版本而异，从而实现受控比较。通过将技能创建、测试和选择嵌入到RL训练循环中，以接近零的额外rollout成本达成技能与策略的调和进化。
- **关键技术细节**：
  1. **组内技能测试（Within-Group Skill Testing）**：  
     每个GRPO组中的不同rollout被分配不同的技能版本（通过Thompson采样决定），从而在不增加总rollout数量的前提下，同时服务于三个目的：策略梯度更新（GRPO损失）、失败诊断（充填经验储池）、版本评估（更新Beta后验用于接收/拒绝）。目标函数为 `J_RESKILL(θ)`，在原GRPO目标基础上替换静态技能库为版本依赖分配。
  2. **RL-in-the-loop技能创建（Skill Creator Pipeline）**：
     - **经验储池（Experience Reservoir）**：持续收集最新训练中的成功-失败轨迹对，附带版本标签和奖励。
     - **断言驱动的失败分级（Assertion-Based Failure Grading）**：维护动态规则断言集，对储池中所有轨迹统一评估，获得通过率，由LLM分析器产出排名靠前的失败诊断，并更新断言集。
     - **历史信息技能修订（History-Informed Skill Revision）**：根据当前技能库、失败诊断和以往变更的记录，由技能创建设计师提出一组操作（添加/修改/删除），生成候选版本 `S_new`，并通过触发器激活率验证后进入测试。
  3. **带有自适应折扣的Thompson采样（Adaptive Thompson Sampling for Skill Evolution）**：
     - **版本分配**：将新旧技能版本视为两个臂，每个臂的成功概率服从Beta分布；在每个训练步，通过蒙特卡罗采样估计 `Pr(p_new > p_old)` 并裁剪以保持最低探索率，随后为每个rollout独立采样版本。
     - **策略感知后验更新**：随着策略进化，较早的观察会过时。引入折扣权重 `w_t = 1/(1 + n_t/M)`，其中 `n_t` 为该步该臂的样本数，`M` 为记忆参数。每个步后，“伪计数”按 `w_t` 收缩后再累加当前步的观测，实现自适应遗忘：样本多的臂折扣更激进，样本少的臂保留更多历史。
     - **接收/拒绝决策**：在一个进化周期结束时，比较新旧版本在折扣后验下的期望成功概率，若新版本更高则被采纳，否则回退至旧版本。记忆参数 `M` 通过已完成周期的序贯预测似然最大化在线估计，无需手动调节。
- **算法流程**（简化）：
  - 初始化策略、技能库、经验储池、后验等。
  - 对每个训练步：
    1. 采样任务，根据当前Thompson后验分配每个rollout的技能版本。
    2. 生成轨迹，计算奖励，更新策略（通过 `J_RESKILL`）。
    3. 将轨迹存入经验储池。
    4. 更新Beta后验（带折扣）。
    5. 每隔 `K_e` 步（进化频率）进行一次技能进化：接收或拒绝当前候选版本；从经验储池中诊断失败并提出新候选版本；重置后验开始新测试周期。
  - 返回优化后的策略和技能库。

## 6. 论文的主要结论与发现

- **ReSkill在所有基准上一致胜出**，尤其在未见（Unseen）任务上提升最大。例如ALFWorld 8B上ReSkill在未见任务达95.3%，比最强基线SkillRL（82.6%）高12.7%；Search上未见任务提升量为Seen的2.1倍。
- **调和进化的重要性**：自动接受（无测试）导致性能崩溃，低于原始GRPO；均匀分配次优；ReSkill的Thompson采样调和了探索与利用，避免技能冲突。
- **技能生命周期自动演化**：训练早期生成基础技能，后期修剪已内化的技能，转向精细化技能，证明了技能与策略的协同进化。
- **跨域泛化能力**：在ALFWorld上训练的策略在测试时仅通过技能创建即可适应ScienceWorld，表明调和训练发展了跟随新技能的能力。
- **鲁棒性**：即使使用较弱模型（Qwen3-4B）创建技能，ReSkill仍优于所有现有RL基线。
